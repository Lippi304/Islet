<objective>
Fix the first-ever Release-build launch crash of Islet. The embedded
`MediaRemoteAdapter.framework` fails Library Validation under Hardened Runtime
(app + framework both ad-hoc signed → different/absent Team IDs → dyld rejects the
mapping). On macOS 26/27 (Tahoe) this is enforced strictly, crashing every Release
build at launch.

The fix is the documented remedy for embedding a third-party framework under
Hardened Runtime: add a single entitlement, `com.apple.security.cs.disable-library-validation`,
via a new entitlements plist wired into the Islet target through `project.yml`
(the xcodegen source of truth), then regenerate the project.

Purpose: Unblock ALL Release builds and thus on-device Release verification and
distribution. This is a prerequisite for Phase 13 (notarization), where the same
framework will need this entitlement anyway.

Output: `Islet/Islet.entitlements`, updated `project.yml`, regenerated
`Islet.xcodeproj`, a passing Release build, and on-device launch confirmation.
</objective>
