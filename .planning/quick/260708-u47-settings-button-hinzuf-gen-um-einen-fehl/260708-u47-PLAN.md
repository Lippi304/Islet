<objective>
Add a "Save Diagnostic Report…" button to the Settings window that snapshots the app's
current diagnostic state (version, OS, hardware, feature toggles, license/trial summary,
Now Playing bridge health) into a plain-text string and lets the user save it via
NSSavePanel — so a user hitting a bug can attach one file to a report instead of
describing their setup by hand.

This is a point-in-time SNAPSHOT, not a logging system: no new persistent log file, no
background collection pipeline, nothing written unless the user clicks the button. It
reuses state that already exists in the app (LicenseState.shared.status, the @AppStorage-
backed activity toggles already bound in SettingsView, LaunchAtLogin.isEnabled, and
NotchWindowController's existing `nowPlayingState.isHealthy` flag — the same flag that
already drives the "Now Playing nicht verfügbar" UI text, NOW-03).

Purpose: give the user (and future bug reports) a one-click, no-guesswork diagnostic
snapshot without inventing a logging subsystem this beginner-maintained app doesn't have.
Output: `Islet/Diagnostics.swift` (testable pure report-assembly logic), its unit tests,
and the Settings UI wiring to trigger + save it.
</objective>

  <name>Task 1: Create DiagnosticReport — pure, testable report-assembly logic</name>
  <files>Islet/Diagnostics.swift, IsletTests/DiagnosticReportTests.swift</files>
  <behavior>
    - `DiagnosticReport.licenseSummary(for: .trial(daysRemaining: 2))` → `"Trial (2 days remaining)"`
    - `DiagnosticReport.licenseSummary(for: .trial(daysRemaining: 1))` → `"Trial (1 day remaining)"` (singular)
    - `DiagnosticReport.licenseSummary(for: .trialExpired)` → `"Trial expired"`
    - `DiagnosticReport.licenseSummary(for: .licensed)` → `"Licensed"`
    - `DiagnosticReport.text(...)` with all inputs supplied contains one line per section:
      app version, macOS version, hardware model, `"License: Licensed"`,
      `"Launch at Login: on"`, `"Charging: on"`, `"Now Playing: off"`, `"Devices: on"`,
      `"Accent index: 2"` — reflecting exactly the booleans/ints passed in.
    - `nowPlayingHealthy: false` → text contains `"Now Playing bridge: unavailable"`.
    - `nowPlayingHealthy: nil` → text contains `"Now Playing bridge: unknown"`.
    - `DiagnosticReport.hardwareModel()` returns a non-empty string on this machine.
  </behavior>
  <action>
Create `Islet/Diagnostics.swift` with a new `enum DiagnosticReport` (internal access, no
`private`, so `@testable import Islet` can reach it — mirrors this repo's existing
pure-seam convention, e.g. NowPlayingPresentation.swift). It has three static functions:

1. `licenseSummary(for status: LicenseStatus) -> String` — switches on the three cases per
   the `<behavior>` block above. This is the ONLY place license state becomes user-facing
   text for this feature; it must never read or format a raw license key (there isn't one
   in `LicenseStatus` to leak — keep it that way).

2. `hardwareModel() -> String` — reads `hw.model` via `sysctlbyname` (two-call pattern: an
   initial call with a nil buffer to size the result, then a second call into an allocated
   `[CChar]` buffer of that size), returning `"unknown"` if either call fails rather than
   crashing or force-unwrapping. `import Darwin` for `sysctlbyname` (do not rely on
   Foundation re-exporting it).

3. `text(licenseStatus: LicenseStatus, launchAtLogin: Bool, chargingEnabled: Bool,
   nowPlayingEnabled: Bool, deviceEnabled: Bool, accentIndex: Int, nowPlayingHealthy:
   Bool?) -> String` — assembles the full plain-text report as a multi-line string:
   a title line, an ISO8601 generated-at timestamp (`ISO8601DateFormatter().string(from:
   Date())`), then labeled lines for: App Version (call `SettingsView.versionString` —
   already exists, do not duplicate its logic), macOS (`ProcessInfo.processInfo
   .operatingSystemVersionString`), Hardware Model (`hardwareModel()`), License
   (`licenseSummary(for:)`), Launch at Login (`"on"`/`"off"`), then an "Activities:"
   block with Charging/Now Playing/Devices (`"on"`/`"off"`) and Accent index (the raw
   int), and finally a "Now Playing bridge:" line that reads `"available"` /
   `"unavailable (MediaRemote bridge not responding)"` / `"unknown (island not running)"`
   for `.some(true)` / `.some(false)` / `nil` respectively.

   Everything the function needs is a PARAMETER except the three one-line system calls
   (timestamp, `SettingsView.versionString`, `ProcessInfo`, `hardwareModel()`) — this keeps
   the assembly logic itself fully unit-testable without booting the real app.

Then create `IsletTests/DiagnosticReportTests.swift` (XCTest, `@testable import Islet`,
mirroring the style of `IsletTests/LicenseStateTests.swift`) covering every bullet in
`<behavior>` above as a separate test method.
