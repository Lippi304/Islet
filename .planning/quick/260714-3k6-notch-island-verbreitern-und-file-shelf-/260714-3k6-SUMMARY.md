# Quick Task 260714-3k6: Notch Island Widen + File-Shelf Consolidation to Tray Summary

**Expanded island panel widened 360→420pt and the file-shelf strip gated to render only inside the Tray tab, with two on-device UAT gap-closure rounds fixing media-player overflow/spacing and empty-state camera clearance**

## Performance

- **Started:** 2026-07-14T02:39:42+02:00 (first task commit)
- **Completed:** 2026-07-14T03:09:54+02:00 (final gap-closure commit)
- **Duration:** ~30 min
- **Tasks:** 2 planned auto tasks + 2 gap-closure fixes (post-checkpoint, pre-approval)
- **Files modified:** 2 (`NotchPillView.swift`, `NotchWindowController.swift`)

## Accomplishments

- Widened the expanded island from 360pt to 420pt (single constant change; every presentation deriving from `expandedSize` — Home, Calendar, Weather, Now-Playing, Tray — grew automatically, still centered via `NotchGeometry`'s existing generic centering)
- Replaced 5 duplicated `shelfVisible: shelfViewState.isVisible` call sites with one named `shelfStripVisible` gate (always `false`), removing the shelf-strip band from ever appearing under Home/Calendar/Weather/Now-Playing — it now only renders inside the dedicated Tray tab
- Simplified `NotchWindowController.visibleContentZone()` in lockstep (no more shelf-height term for non-Tray presentations), keeping the click-through hit-test geometry from drifting out of sync with the render side (the exact CR-01 regression class this project has hit before)
- Two on-device UAT rounds surfaced and fixed real regressions from the width bump:
  - Now-Playing content (equalizer bars specifically) rendered ~5pt past the visible rounded shape's edge, because `NotchShape`'s side walls sit at a constant `topCornerRadius` (24pt) inset from each edge, independent of panel width, while the old horizontal padding was only 19pt
  - `homeEmptyState` used a bare `.padding(.top, 24)` instead of the shared `Self.cameraClearance` (42) every sibling presentation uses, sitting its icon/text noticeably closer to the camera
  - After the wall-clearance fix, the player still read as "spread out" because its internal `Spacer`-driven gaps stretched to fill the full ~368pt remaining row width — fixed by capping the content card to a fixed 322pt width (matching the old panel's own content width) instead of relying on padding alone

## Files Created/Modified

- `Islet/Notch/NotchPillView.swift` - `expandedSize` widened to 420×144; `shelfStripVisible` gate added (replacing `isTrayPresentation`); outer body `.frame` shelf-height addition removed; 5 `blobShape` call sites (homeEmptyState, calendarFullView, weatherFullView, mediaExpanded, mediaUnavailable) now pass `shelfVisible: shelfStripVisible`; `mediaExpanded`'s content capped to `.frame(maxWidth: 322)`; `homeEmptyState`'s top padding switched to `Self.cameraClearance`
- `Islet/Notch/NotchWindowController.swift` - `visibleContentZone()` simplified: removed `isTrayPresentation`/`shelfHeight` locals and the `+ shelfHeight` term from `contentSize`, matching the render-side gate

## Decisions Made

- 420pt reuses the width already established for `onboardingSize` — no new magic number introduced for the panel width
- The shelf gate is one boolean (`shelfStripVisible`), not a parameter threaded through `blobShape`'s signature — keeps the diff surgical and matches this file's existing single-source-of-truth convention (`cameraClearance`, `switcherContentHeight`)
- `trayFullView` and `onboardingCarousel` were deliberately left untouched (they already pass `shelfVisible: false` for unrelated reasons)
- After round-1's wall-clearance padding fix (19→26) still read as "spread out" on-device, the fix was redirected from padding to a `.frame(maxWidth: 322)` cap on the content card — this simultaneously restores 360pt-era density AND clears the 24pt wall inset (322 + ~49pt margin per side, comfortably inside the 420pt outer frame), so the separate horizontal padding became unnecessary

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug, found via on-device UAT round 1] Media content overflowed past the rounded panel edge**
- **Found during:** Checkpoint (Task 3) round 1, user on-device test
- **Issue:** `mediaExpanded`'s `.padding(.horizontal, 19)` sat inside `NotchShape`'s constant `topCornerRadius` (24pt) wall inset (confirmed by reading `NotchShape.swift`'s path construction — the left/right walls sit at a fixed `topCornerRadius` offset from each edge across the whole content-height range, independent of total panel width), so the equalizer bars (pinned to the row's trailing edge via `Spacer`) rendered ~5pt past the visible rounded shape on both sides
- **Fix:** Bumped `.padding(.horizontal, 19)` → `26` (topCornerRadius + 2pt margin)
- **Files modified:** `Islet/Notch/NotchPillView.swift`
- **Verification:** `xcodebuild build` BUILD SUCCEEDED; user re-tested on-device round 2, confirmed edge overflow visually resolved
- **Committed in:** `bc1e73a` (superseded by round 2's `db11d72`, see below)

**2. [Rule 1 - Bug, found via on-device UAT round 1] Home empty-state sat too close to the camera**
- **Found during:** Checkpoint (Task 3) round 1, user on-device test
- **Issue:** `homeEmptyState` used a bare `.padding(.top, 24)` instead of the shared `Self.cameraClearance` (42) every other switcher-row presentation (`mediaExpanded`, `calendarFullView`, `weatherFullView`, `trayFullView`) already uses — its icon/text sat noticeably higher/closer to the camera than the playing-state view
- **Fix:** Changed `.padding(.top, 24)` → `.padding(.top, Self.cameraClearance)`
- **Files modified:** `Islet/Notch/NotchPillView.swift`
- **Verification:** `xcodebuild build` BUILD SUCCEEDED; user re-tested on-device round 2, confirmed clearance now matches the playing-state view
- **Committed in:** `bc1e73a`

**3. [Rule 1 - Bug, found via on-device UAT round 2] Media player read as "spread out" at the wider width**
- **Found during:** Checkpoint (Task 3) round 2, user on-device test
- **Issue:** The round-1 wall-clearance padding (26) still let `mediaExpanded`'s internal `Spacer`-driven gaps (album art ↔ title/artist ↔ equalizer bars, transport-button spacing) stretch to fill the full ~368pt remaining row width at 420pt panel width, making the player feel loose/thin compared to its 360pt-era density
- **Fix:** Replaced `.padding(.horizontal, 26)` with `.frame(maxWidth: 322)` on the content `VStack` — 322 matches the old 360pt panel's own content width, so every internal `Spacer`-driven gap collapses back to the original density; centered by `blobShape`'s existing `alignment: .top` outer frame (~49pt margin each side), which also clears the 24pt wall inset, superseding the need for the separate horizontal padding
- **Files modified:** `Islet/Notch/NotchPillView.swift`
- **Verification:** `xcodebuild build` BUILD SUCCEEDED; user re-tested on-device round 3, confirmed "Passt" (fits/good)
- **Committed in:** `db11d72`
