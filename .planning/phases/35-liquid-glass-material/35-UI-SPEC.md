| Role | Value | Usage |

| Destructive | Not applicable | No destructive actions in this phase |

Accent reserved for: the displacement-map edge band only (the outer `borderWidth` fraction of each shape, where `feDisplacementMap`'s per-channel offset produces the rainbow-fringe warp). Never applied to text, icons, buttons, or the settings sidebar selection highlight.

## Material / Shader Contract (this phase's real design contract)

### Island shell (pill, expanded island, all 3 wings) — full effect

Applies as a modifier stack on the SAME shape node that already carries `.matchedGeometryEffect(id: "island", in: ns)` at all 4 existing `.fill(islandFill)` call sites (`NotchPillView.swift` ~1532, 1669, 1734, 2061) — never a new sibling/wrapper view (ROADMAP Success Criterion #2, WR-02 regression class).

Render order (back to front): `gradientMaterial` fill → `.distortionEffect()` Metal shader (warp + chromatic fringe) → frost overlay (`backgroundOpacity`-equivalent) → foreground content (undistorted, per D-03).

Starting shader parameters (on-device tuning starting points per CONTEXT.md Claude's Discretion — NOT final; the reference component's 200–400pt web defaults do not map 1:1 to Islet's 32–420pt canvas, so these are pre-scaled by canvas-size ratio, then must be verified/adjusted during execution):

| Parameter | Collapsed pill (32pt tall) | Expanded island / wings | Notes |
|-----------|---------------------------|--------------------------|-------|
| `borderRadius` | `NotchShape`'s actual radius | `NotchShape`'s actual radius | Never a free value — read from the shape, matches D-01 |
| `borderWidth` (edge-band fraction) | 0.15 | 0.10–0.12 | Proportionally wider on the smaller pill so the band stays visible at all (web default 0.07 is too thin at 32pt) |
| `brightness` (map center brightness) | 50 | 50 | Unscaled — this is a 0–100 tone value, not a distance |
| `blur` (pre-displacement blur, pt) | 2–3 | 6–8 | Scaled down from web's 11 by canvas-size ratio |
| `distortionScale` (overall warp strength) | −4 to −6 | −10 to −16 | Scaled down from web's −180; MUST stay subtle enough that pill/wing text and icons remain legible |
| `redOffset` / `greenOffset` / `blueOffset` | 0 / ~0.5 / ~1 (near-off per D-04) | 0 / 1–1.5 / 2–3 | Scaled down from web's 0/10/20; drives chromatic-fringe width |
| `saturation` | 1.0 | 1.05–1.1 | Backdrop saturation multiplier |
| `backgroundOpacity` (frost) | 0.04 | 0.06–0.08 | Subtle — must not wash out foreground content contrast |

> **Implementation note (Plan 35-02, resolves a documentation/implementation divergence found during plan review):** `brightness` is NOT implemented as a separate runtime shader parameter or `LiquidGlassParameters` field. Plan 35-02's Metal function folds the map-center-brightness effect into the edge-falloff curve (the `smoothstep`-based transition from full warp at the edge to near-zero at the interior) — that falloff already asymptotes toward the interior the same way a mid-tone `brightness` value tempered the center of the original SVG displacement map, so a distinct field would be redundant. This row stays in the table for traceability to the reference technique's props, but has no corresponding Swift/Metal parameter.

Distortion strength scales with view-state size per D-04: the collapsed pill uses the reduced/near-off column above (barely visible at 32pt anyway); the expanded island and wings use the full column. Whether this is a continuous size-driven interpolation or a binary collapsed/expanded switch is Claude's Discretion (execution-time call, whichever is cheaper to build correctly) — either satisfies this contract as long as the collapsed state reads as visibly subtler than expanded.

**Hard constraints (from CONTEXT.md decisions, non-negotiable):**
- D-01: `.distortionEffect(_:maxSampleOffset:)` Metal shader — not `.glassEffect()`/`NSGlassEffectView` (no deployment-target bump).
- D-02: `gradientMaterial`'s existing stops/direction are the base — the shader composes on top, never replaces them.
- D-03: distortion applies to the fill layer only — album art, equalizer bars, text, and icons stay crisp/undistorted.
- D-04: distortion strength scales down for the collapsed pill vs. full strength expanded.
- Success Criterion #2: applied at the existing 4 `.fill(islandFill)` sites carrying `matchedGeometryEffect(id: "island")` — no new view in the hierarchy.

## Interaction / Animation Contract

- The expand↔collapse spring morph (Phase 25 baseline, its expand-animation regression fixed as this phase's prerequisite per D-07) is assumed correct going in — Phase 35 must not regress it. The material composites cleanly mid-morph: no visual artifacts, no dropped frames, verified on-device in the real never-focused panel (ROADMAP Success Criterion #4).
- `.distortionEffect()` warps only Islet's own rendered fill layer — it does NOT bend the live desktop behind the transparent `NSPanel` (there is no "see-through glass" effect; confirmed constraint from `reference-GlassSurface.md`). Do not design around or promise a live-background-warp effect.
- Switching MaterialStyle in Settings (Gradient / Solid Black / Liquid Glass) live-updates the island shell without requiring a relaunch, matching the existing Phase 27 picker behavior.
- Idle-CPU cost: if the shader has non-trivial idle cost while the island is visible but static, consider the existing "idle-CPU-gated" animation pattern used by the equalizer bars (Phase 4 precedent) — flagged as a watch-item, not a hard requirement, per CONTEXT.md code-context notes.
