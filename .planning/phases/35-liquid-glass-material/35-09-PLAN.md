<objective>
Implement the round-3 design pivot from `35-CONTEXT.md` D-12/D-13/D-14/D-15: restructure the Liquid Glass material so a genuine solid dark "frost" layer (reusing the existing `Self.gradientMaterial` near-opaque black constant) is the true, always-present visual base, with the live-blurring `.ultraThinMaterial` backdrop only revealed through a narrow, opacity-ramped band right at the rounded edge — the exact inverse compositing order from round 2's rejected attempt, where the bright `.ultraThinMaterial` itself carried the opacity ramp with no dark layer underneath.

Round 1 (35-05) rejected an opaque `gradientMaterial` base with no visible transparency. Round 2 (35-06/35-07/35-08) rejected a raw `.ultraThinMaterial` base because it has no inherent dark tint and reads as uniformly bright, tracking whatever's behind the notch. This plan fixes round 2's root cause without touching the Metal shader files at all: `liquidGlassEdgeFalloff`/`liquidGlassEdgeOpacity` are reused byte-for-byte, only WHICH SwiftUI layer consumes them changes (the frost, not the material), plus the Swift-side `LiquidGlassParameters` numeric starting points are retuned.

Purpose: Give Plan 35-10's on-device UAT checkpoint a build where the island reads as dark glass (matching `reference-transparency-target.png`) rather than a bright, backdrop-tracking panel.
Output: `LiquidGlassParameters` retuned per D-14/D-15; `islandFill`'s `.liquidGlass` branch returns the D-12 frost base; `liquidGlassEffectLayer` composites frost-over-material instead of material-with-its-own-ramp.
</objective>
