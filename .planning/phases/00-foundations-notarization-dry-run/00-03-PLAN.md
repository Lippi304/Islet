<objective>
Write the re-runnable release pipeline as a single commented `scripts/release.sh`: archive → export the `.app` → codesign with hardened runtime → build a `.dmg` (via `hdiutil`) → sign the DMG → submit to `notarytool` → `stapler staple` → `spctl` verify. The Developer-ID identity and notary credentials are PLACEHOLDER variables, clearly marked, so the user fills them in once and re-runs the script UNCHANGED at Phase 6. With no real cert, the script signs ad-hoc (`codesign -s -`) and cleanly SKIPS (does not fail) the notarize/staple steps. Also add the `.gitignore` for Xcode artifacts and a beginner-facing `docs/RELEASE.md`. (Implements APP-04, success criterion #4.)

Purpose: The single biggest first-timer footgun is a broken release toolchain discovered at ship time. D-02 requires this proven as a repeatable script NOW, on a hello-world build, not hand-typed later. The real `notarytool submit` + `stapler staple` + clean-Mac open are deliberately DEFERRED to Phase 6 (the Apple Developer account is deferred per D-01) — this plan PREPARES and DOCUMENTS them; it does not run them.
Output: `scripts/release.sh`, `.gitignore`, `docs/RELEASE.md`.

This plan WRITES the script and docs only (no source-code dependency, no built app needed). Plan 04 actually RUNS the script against the finished app and performs the local Gatekeeper demo.

CRITICAL constraints (from RESEARCH + CONTEXT):
- Use `hdiutil`, NOT `create-dmg` (create-dmg is not installed; hdiutil ships with macOS — RESEARCH Alternatives Considered).
- Stage the app with `ditto`, NOT `cp -r` (cp -r corrupts framework symlinks — RESEARCH anti-pattern).
- Use `xcrun notarytool`, NOT the removed `altool`.
- `--options runtime --timestamp` is mandatory for notarization (RESEARCH Pitfall 4).
- NO secrets in the repo: the script references a keychain notary PROFILE NAME only; Apple ID / Team ID / Developer ID are placeholders the user fills in locally (RESEARCH Security Domain).
</objective>

Create `scripts/release.sh` — a single Bash script (`#!/usr/bin/env bash`, `set -euo pipefail`) implementing the full pipeline from RESEARCH Pattern 4. EVERY step must be commented for a first-time programmer (what it does + why). The Developer-ID / notary parts are gated behind clearly-marked PLACEHOLDER variables; when the placeholders are still at their default, the script signs ad-hoc and SKIPS (does not fail) notarize/staple with a clear message.

Write the script with these exact, ordered sections:

1. Header + config (PLACEHOLDERS at the very top, loudly marked):
```
#!/usr/bin/env bash
set -euo pipefail

# ============================================================================
# Islet release pipeline:  archive -> sign -> dmg -> notarize -> staple
# ----------------------------------------------------------------------------
# PHASE 0: runs end-to-end EXCEPT real notarization (Apple Developer account is
# deferred — D-01). With the placeholders below unfilled, the script signs
# AD-HOC for local testing and SKIPS notarize/staple with a clear message.
#
# PHASE 6: fill in the two PLACEHOLDER variables below, then re-run this script
# UNCHANGED to produce a real notarized + stapled DMG.
# ============================================================================

# >>> FILL THESE IN AT PHASE 6 (leave as-is for the Phase-0 dry run) >>>
DEVELOPER_ID="__DEVELOPER_ID__"     # e.g. "Developer ID Application: Your Name (TEAMID)"
NOTARY_PROFILE="__NOTARY_PROFILE__" # e.g. "islet-notary"  (see docs/RELEASE.md)
# <<< FILL THESE IN AT PHASE 6 <<<

SCHEME="Islet"
APP_NAME="Islet"
VOL_NAME="Islet"
ARCHIVE_PATH="build/${APP_NAME}.xcarchive"
EXPORT_DIR="build/export"
APP_PATH="${EXPORT_DIR}/${APP_NAME}.app"
DMG_DIR="build/dmgroot"
DIST_DIR="dist"
DMG_PATH="${DIST_DIR}/${APP_NAME}.dmg"
```

2. Clean + archive (xcodebuild):
```
rm -rf build dist
mkdir -p "${EXPORT_DIR}" "${DIST_DIR}"

# Archive a Release build of the app.
xcodebuild -scheme "${SCHEME}" -configuration Release \
  archive -archivePath "${ARCHIVE_PATH}"
```

3. Export the .app out of the archive (read it from the archive's Products, with ditto):
```
# The built .app lives inside the archive at Products/Applications.
ditto "${ARCHIVE_PATH}/Products/Applications/${APP_NAME}.app" "${APP_PATH}"
```

4. Sign the .app — Developer ID if filled, else ad-hoc fallback (D-03). Hardened runtime + timestamp mandatory for notarization (Pitfall 4):
```
if [ "${DEVELOPER_ID}" = "__DEVELOPER_ID__" ]; then
  echo "-> No Developer ID set: AD-HOC signing for local dry-run (D-03)."
  codesign --force --deep --sign - "${APP_PATH}"
else
  echo "-> Signing with Developer ID + hardened runtime."
  codesign --force --options runtime --timestamp \
    --sign "${DEVELOPER_ID}" "${APP_PATH}"
fi
codesign --verify --verbose "${APP_PATH}"
```

5. Build the DMG with hdiutil — stage with ditto, NOT cp -r (corrupts framework symlinks):
```
rm -rf "${DMG_DIR}" && mkdir -p "${DMG_DIR}"
ditto "${APP_PATH}" "${DMG_DIR}/${APP_NAME}.app"
hdiutil create -volname "${VOL_NAME}" \
  -srcfolder "${DMG_DIR}" -ov -format UDZO "${DMG_PATH}"
```

6. Sign the DMG too (only when a real Developer ID is set):
```
if [ "${DEVELOPER_ID}" != "__DEVELOPER_ID__" ]; then
  codesign --force --options runtime --timestamp \
    --sign "${DEVELOPER_ID}" "${DMG_PATH}"
fi
```

7. Notarize + staple — DEFERRED, placeholder-gated (D-01/D-02). Skip cleanly when unset:
```
if [ "${DEVELOPER_ID}" = "__DEVELOPER_ID__" ] || [ "${NOTARY_PROFILE}" = "__NOTARY_PROFILE__" ]; then
  echo "--------------------------------------------------------------"
  echo "SKIPPING notarize + staple — placeholders not filled (Phase 6 step)."
  echo "Phase-0 dry run complete: ${DMG_PATH} (ad-hoc signed, NOT notarized)."
  echo "To finish at Phase 6: fill DEVELOPER_ID + NOTARY_PROFILE, re-run this script."
  echo "--------------------------------------------------------------"
  exit 0
fi

# ---- Phase 6 real run (these execute only once placeholders are filled) ----
xcrun notarytool submit "${DMG_PATH}" --keychain-profile "${NOTARY_PROFILE}" --wait
xcrun stapler staple "${DMG_PATH}"
spctl --assess -vvv --type install "${DMG_PATH}"   # expect: accepted, Notarized Developer ID
echo "-> Notarized + stapled DMG ready: ${DMG_PATH}"
```

Then make it executable: `chmod +x scripts/release.sh`.

Security (RESEARCH Security Domain): the script must reference a keychain PROFILE NAME (`NOTARY_PROFILE`) only — it must NOT contain an Apple ID password, app-specific password, or any plaintext secret. The keychain profile is created once by the user via `xcrun notarytool store-credentials` (documented in RELEASE.md). Do NOT hardcode credentials.

Comment each step explaining what it does and why — this script is also teaching material for a first-time programmer (CONTEXT.md specifics).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| developer machine → Apple Notary service | At Phase 6 the script authenticates to Apple's notary service via a keychain-stored credential profile. The credential never appears in the repo or script. |
| repo → distributable artifact | The script transforms source into a signed/notarized DMG that crosses the boundary to end users; its integrity is what Gatekeeper later enforces. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-00-06 | Information Disclosure | Notary credentials leaked into repo/script | mitigate | Script references a keychain PROFILE NAME (`NOTARY_PROFILE`) only; Apple ID / Team ID / Developer ID are placeholders the user fills locally; credentials created via `notarytool store-credentials` live in the keychain; `.gitignore` excludes build/dist artifacts. Acceptance criteria assert no plaintext password is present. HIGH-impact threat fully mitigated. |
| T-00-07 | Tampering | Distributed binary altered in transit / not trustworthy | mitigate | Hardened-runtime signing (`--options runtime --timestamp`) + Developer-ID sign + notarize + staple chain is scripted; Gatekeeper enforces at the user's machine. Phase-0 demonstrates the BLOCK on an unsigned build locally (Plan 04). No high-severity tampering path left unmitigated for the eventual release. |
| T-00-08 | Repudiation / Integrity | Script silently produces an unsigned/un-notarized artifact mistaken for a release | mitigate | The placeholder-gated skip prints a LOUD message that the DMG is ad-hoc-signed and NOT notarized, so a Phase-0 dry-run artifact can never be confused with a shippable build. Low severity. |
