  <action>
    Add `private func refreshCalendarMonth()`: calls `calendarService.fetchMonth(containing:
    calendarViewState.visibleMonth) { [weak self] events in self?.calendarViewState.monthEvents
    = events }` — the SAME `calendarService` property `refreshCalendar()` already uses, never a
    second instance (CALVIEW-04).

    Add `private func handleSwitcherSelect(_ view: SelectedView)`: wraps the mutation in
    `withAnimation(.spring(response: springResponse, dampingFraction: springDamping))` and
    sets `viewSwitcherState.selectedView = view`, `shelfViewState.forcedByTray = (view ==
    .tray)` (D-02's Tray reconciliation — force-reveals an empty shelf without adding a
    dedicated resolver case), and when `view == .calendar` also resets
    `calendarViewState.selectedDay = Date()`, `calendarViewState.visibleMonth = Date()`,
    `calendarViewState.monthEvents = nil` (Pitfall 4 — avoids a stale-month flash; D-07: today
    selected by default on open). Call `renderPresentation()` inside the same animation block
    (mirrors `presentTransientChange()`'s existing pattern), then call `syncClickThrough()`
    after, and if `view == .calendar` call `refreshCalendarMonth()`.

    Add `private func handleCalendarMonthChange(_ delta: Int)`: computes `Calendar.current.date(
    byAdding: .month, value: delta, to: calendarViewState.visibleMonth)`, guards against nil
    (no-op on failure, never crash), sets `calendarViewState.visibleMonth` to the new month,
    sets `calendarViewState.monthEvents = nil`, then calls `refreshCalendarMonth()` (D-08:
    prev/next month navigation).

    Add `private func handleCalendarDaySelect(_ day: Date)`: sets `calendarViewState.selectedDay
    = day`. No spring animation needed (no shape/size change, just content).

    Add `private func handleQuickAdd(_ kind: QuickAddKind, title: String)`: reads `let day =
    calendarViewState.selectedDay`. For `.event`: calls `calendarService.createEvent(title:
    title, start: day, end: day.addingTimeInterval(3600), completion: { [weak self] _ in
    self?.refreshCalendarMonth() })` (1-hour default duration starting at the selected day — no
    time picker exists per UI-SPEC's Copywriting Contract, so this is the deliberate lightweight
    default; refreshing afterward makes a newly-created event appear in the day list
    immediately). For `.reminder`: calls `calendarService.createReminder(title: title, dueDate:
    day, completion: { _ in })` — no day-list refresh needed since reminders have no rendering
    surface in this phase (CALVIEW-03 is create-only for reminders).

    In `makeRootView(theme:)`, add the 6 new arguments to the `NotchPillView(...)` call:
    `viewSwitcherState: viewSwitcherState`, `calendarViewState: calendarViewState`,
    `onSwitcherSelect: { [weak self] view in self?.handleSwitcherSelect(view) }`,
    `onCalendarMonthChange: { [weak self] delta in self?.handleCalendarMonthChange(delta) }`,
    `onCalendarDaySelect: { [weak self] day in self?.handleCalendarDaySelect(day) }`,
    `onQuickAdd: { [weak self] kind, title in self?.handleQuickAdd(kind, title: title) }` —
    placed alongside the existing `onShelfItem*`/`onOnboarding*` closures.
  </action>
