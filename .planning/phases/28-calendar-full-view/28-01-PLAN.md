<objective>
Build the pure, framework-free foundation for Phase 28: the day/month-bucketing math the
calendar grid needs, and the two new small `@Published` state carriers
(`ViewSwitcherState`/`SelectedView`, `CalendarViewState`/`QuickAddKind`) that later plans wire
into the view and controller. This is the "interface-first" wave — nothing here touches
AppKit, EventKit, or SwiftUI; Plans 02/03/04 compile against the types and functions defined
here.

Note on scope: `IslandPresentation`'s new `.calendarExpanded` case and `resolve(...)`'s new
`selectedView` parameter are deliberately NOT part of this plan — they must land in the SAME
plan/task as `NotchPillView.body`'s exhaustive `switch presentation` gaining its new arm (Swift
requires an exhaustive switch; adding the enum case here without the matching view arm would
break the whole project's build). That pairing is Plan 03 Task 2. `SelectedView` is defined
HERE (in `ViewSwitcherState.swift`) specifically so Plan 03's `resolve(...)` extension can
reference an already-existing type without a forward dependency.

Purpose: keep the calendar math deterministically unit-testable, per `CalendarGlance.swift`'s
existing `now:`-explicit-parameter discipline (T-14), and give Plans 02-04 real, compilable
contracts to build against instead of discovering shapes mid-implementation.
Output: extended `CalendarGlance.swift`, two new state-carrier files, extended
`CalendarGlanceTests.swift`.
</objective>

    `func daysInMonth(for date: Date, calendar: Calendar = .current) -> [Date?]` — returns a
