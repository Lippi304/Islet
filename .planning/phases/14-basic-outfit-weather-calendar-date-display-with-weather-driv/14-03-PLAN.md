    `completion?(locations.last); completion = nil`. Implement
    `locationManager(_:didFailWithError:)`: call `completion?(nil); completion = nil` (D-01: any
    failure is silent omission, never surfaced). This is a THIN wrapper — no persistent
    tracking, no `startUpdatingLocation()`, only the single-shot `requestLocation()` API

    monitoring).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| OS Location/Calendar permission prompt -> app | User-controlled access-control gate (V4); denial must be honored, never bypassed/nagged |
| Subscribed/shared calendar -> EKEvent.title | Untrusted external string (V5), same class as the existing Bluetooth device-name precedent (T-05-01) |
| App -> WeatherKit / EventKit (Apple first-party services) | Standard Apple framework trust boundary, no new third-party data-sharing surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-14-05 | Elevation of Privilege (permission bypass/nag) | `LocationProvider.requestOnce`, `EventKitService.fetchUpcoming` | mitigate | Both settle with `completion(nil)` on any non-authorized status, with NO retry loop and NO re-prompt — verified by the acceptance_criteria grep on `completion(nil)` presence in both files (D-01/D-03) |
| T-14-06 | Information Disclosure / Tampering (untrusted calendar title) | `EventInput.title` sourced from `EKEvent.title` | mitigate | Passed through as a plain `String` only, never interpolated into format/log/shell strings in this plan; render-time bounding (`.lineLimit(1)`/`.truncationMode(.tail)`) is enforced in 14-04's view layer (T-05-01 precedent) |
| T-14-07 | Information Disclosure | Location coordinates sent to WeatherKit | accept | WeatherKit is Apple's own first-party service under the existing Developer Program agreement — no additional third-party data-sharing surface introduced |

No package-manager installs in this plan (WeatherKit/EventKit/CoreLocation/AppKit are all
first-party system frameworks) — the Package Legitimacy Gate does not apply.
</threat_model>
