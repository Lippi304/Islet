<objective>
Build the two pure, Foundation-only classification/selection seams that every later Phase-14
plan depends on: `WeatherCategory.from(_:)` (D-06's 4-category mapping) and
`nextRelevantEvent(events:now:)` (D-04's "next upcoming/in-progress today, else tomorrow's
first, else nothing" selection). Both are total functions with zero system-framework calls
(no WeatherKit network call, no EventKit query) so they are unit-tested deterministically in
milliseconds, mirroring `Islet/Notch/DeviceActivity.swift`'s existing pure-seam convention.

Purpose: isolates the riskiest classification logic from the fragile, permission-gated,
async externals (WeatherKit/EventKit) built in 14-03, so those services stay thin glue that
just calls into these already-proven functions.
Output: `Islet/Weather/WeatherCategory.swift`, `Islet/Calendar/CalendarGlance.swift`, and their
passing XCTest suites.

    - Given zero events today AND zero events tomorrow (only an event 3 days out),
