## Spacing Scale

This codebase's existing convention (verified in `SettingsView.swift`) does not follow a strict 4pt/8pt grid — it uses native `Form`/HIG defaults plus a few explicit values. Documenting as-is; do not invent a new grid for this restructuring-only phase.

| Token | Value | Usage (existing, verified in file) |
|-------|-------|-------|
| Sidebar row padding | 8px (container), 6pt vertical / 10pt horizontal (row) | `SidebarSection` button rows, line 120-121 — unchanged by this phase |
| Section content padding | 20px | `.padding(20)` on every detail-pane `Form` (generalSection, aboutSection, systemSection) — reuse for all 5 new sections |

| Swatch row spacing | 10px | Accent-color circle row (`swatchRow`, line 453) — unaffected, moves as-is into Appearance |

This phase adds zero new spacing values — it relocates existing `Section { ... }` blocks (Activities, Fullscreen, Weather, Diagnostics) verbatim from the monolithic `generalSection` Form into their own top-level `Form.padding(20)` detail views, and wraps each in a `ScrollView` only where content can overflow the fixed 380pt window height (see Interaction Contract below).

## Interaction Contract — Scroll Fix (SETTINGS-02)

- Window stays fixed at `520×380` (D-05, locked) — no resizing, no growing taller.
- Root cause (for planner/executor): the current single monolithic `generalSection` `Form` has no explicit `ScrollView` — SwiftUI's `Form` alone does not reliably become scrollable inside a fixed-height `NavigationSplitView` detail pane on this setup once content height exceeds the pane, which is why Weather/Diagnostics (positioned last, after Activities' 8 toggles + hints) are currently unreachable.
- Fix mechanism: wrap each detail-pane section's `Form` in an explicit `ScrollView { Form { ... } }` (or equivalent) so any section whose content exceeds the available height scrolls internally. Apply consistently to all 7 sections (5 new + Workspace + About) for uniform behavior, even though only Activities is expected to need it in practice (D-05: Fullscreen/Weather/Diagnostics are short enough to normally fit without scrolling).
- Activities remains the tallest section (Launch at Login + 8 toggles + up to 2 conditional permission-hint `Text` rows) — this is the section that must be verified scrollable end-to-end (its own last toggle, "Automatically Check for Updates," must be reachable and visible after scrolling).

## Sidebar Structure (locked, 51-CONTEXT.md D-06)

Final order, top to bottom: **Activities → Appearance → Fullscreen → Weather → Diagnostics → Workspace → About**
