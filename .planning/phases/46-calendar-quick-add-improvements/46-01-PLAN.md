<objective>
Add the real date+time picker to `QuickAddPopover` (CALVIEW-05, D-01 through D-05, D-07) and the
pure default-time helper it seeds from. This plan widens the popover's `onSubmit` contract to carry
real `Date` values but does NOT wire those values into `NotchWindowController`/`CalendarService`
yet — that lockstep-signature wiring, plus the button-placement (CALVIEW-06) and row-padding/island
-growth (CALVIEW-07) changes, is Plan 46-02 (same file, sequenced after this plan by design — see
`files_modified` overlap).

Purpose: `QuickAddPopover` currently has no date/time UI at all — it hardcodes the Event's start to
the selected day and end to start+1hr, and the Reminder's due date to the selected day
(`Islet/Notch/NotchWindowController.swift:1651-1661`). This plan gives the user a real, native
`.compact` time picker per the locked decisions in `46-CONTEXT.md`.

Output: `defaultQuickAddTime(selectedDay:now:)` (pure, tested), `QuickAddPopover` extended with
Starts/Ends/Due rows and auto-follow logic, `onSubmit`/`onQuickAdd` widened to
`(QuickAddKind, String, Date, Date?) -> Void`.
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-46-01 | Tampering | `defaultQuickAddTime(selectedDay:now:)` | accept | Pure derivation from two `Date` inputs the caller already owns (`selectedDay` from `CalendarViewState`, `now` a live system read) — no external/network input, no persistence. |
| T-46-02 | Information Disclosure | `startTime`/`endTime`/`endManuallyEdited` `@State` | accept | Local, transient SwiftUI view state scoped to the popover's own lifetime, never persisted or logged — same trust tier as the existing `title`/`kind` `@State` this plan extends. |
| T-46-03 | Denial of Service | `DatePicker`-driven `Date` values fed into `onSubmit` | accept (deferred to Plan 46-02) | `DatePicker` constrains the value to a valid `Date` by construction (no free-text date parsing) — any downstream EventKit validity constraints (e.g. End before Start) are Plan 46-02's `CalendarService` call-site concern, not this plan's, since this plan does not yet call `CalendarService`. |

No Spoofing / Repudiation / Elevation-of-Privilege entries — this plan adds no new identity, network,
