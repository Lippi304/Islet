## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-46-04 | Tampering | `handleQuickAdd`'s `end ?? start.addingTimeInterval(3600)` fallback | accept | Defensive-only fallback for a case `QuickAddPopover`'s submit call never actually produces (Event always passes a real `endTime`); no attacker-controlled input reaches this path — same trust tier as the removed `day.addingTimeInterval(3600)` it replaces. |
| T-46-05 | Denial of Service | `CalendarService.createEvent` with an End before Start (e.g. user manually sets End earlier than Start after D-04's auto-follow stops) | accept | EventKit itself rejects/normalizes an invalid `EKEvent.endDate < startDate` at save time (existing `CalendarService.createEvent`'s own `completion: (Bool)` failure path, unchanged by this plan) — no new validation gap introduced; same risk tier as the phase 28 original which had the identical unconstrained-range property. |
| T-46-06 | Tampering | `dayListColumn`/`dayEventsList` layout constants (`calendarWidth`, `calendarContentHeight`, padding literals) | accept | Pure static-constant/layout-literal changes, no external input, no new attack surface — same tier as every prior Phase 29/32/44/45 sizing-constant change in this codebase. |

No Spoofing / Repudiation / Information Disclosure / Elevation-of-Privilege entries — this plan adds
no new identity, network, or privilege boundary.
