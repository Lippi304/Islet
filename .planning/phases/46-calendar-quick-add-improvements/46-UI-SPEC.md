# Phase 46 ‚Äî UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Layout Contract

### Date+time picker (CALVIEW-05, D-01 through D-05, D-07)

Lives entirely inside `QuickAddPopover.quickAddContent` (`NotchPillView.swift` ~line 3156) ‚Äî no new view/file.

| Element | Value | Source |
|---------|-------|--------|
| Date field | NONE ‚Äî date is locked to `calendarViewState.selectedDay`, never shown as an editable control | D-01 |
| Time control style | `.datePickerStyle(.compact)` on every time `DatePicker` ‚Äî small native tappable field, minimal vertical footprint | D-02 |
| Event layout | Two separate rows inside the existing `VStack(alignment: .leading, spacing: 8)`: a "Starts" row, then an "Ends" row ‚Äî each its own `HStack` (label + compact `DatePicker`), not one side-by-side row | D-03 |
| Event End auto-follow | Changing Start shifts End to preserve exactly a 1-hour duration, UNTIL the user directly edits End ‚Äî after that, End stops auto-following Start for the rest of that popover session (`@State private var endManuallyEdited = false`, flipped true in End's `.onChange`, checked before Start's `.onChange` writes End) | D-04 |
| Reminder layout | Only ONE time row is shown, labeled **"Due"** ‚Äî the End row is fully absent (not disabled/greyed), matching EventKit's `dueDate` terminology | D-05 |
| Default time (both kinds) | Next full hour if `calendarViewState.selectedDay` is today, else `00:00` ‚Äî computed via a pure helper taking `now`/reference date as an explicit parameter (project convention, no inline `Date()`); applied uniformly to Event Start and Reminder Due | REQUIREMENTS.md CALVIEW-05 + Claude's Discretion (uniform helper) |
| Default End (Event) | Start + 1 hour, recomputed whenever Start changes and End hasn't been manually edited | D-04 |
| Popover open direction | Forced toward the day list (trailing/right) ‚Äî e.g. `.popover(isPresented:attachmentAnchor:arrowEdge: .trailing)` ‚Äî must never expand leftward over the narrow month-grid column | D-07 |
| Popover width | 240pt (was 220pt ‚Äî +20pt to comfortably fit two labeled compact-DatePicker rows without text/control crowding; directional, tune on-device if the compact DatePicker renders wider than expected) | New, Claude's Discretion |

## Verification Notes for Executor

- `endManuallyEdited` must be a `QuickAddPopover`-local `@State`, reset (`= false`) whenever the popover is dismissed/reopened (`isShowing` toggles false‚Üítrue) ‚Äî otherwise a stale "manually edited" flag from a previous quick-add session could suppress the auto-follow behavior on the next one.
- The "next full hour if today, else 00:00" default-time computation MUST take `now`/reference date as an explicit parameter ‚Äî do not call `Date()` inline inside the view or inside the computation helper (hard project convention, `CalendarGlance.swift`/T-14 discipline, reaffirmed for this phase in `46-CONTEXT.md`'s Code Context section).
- `calendarContentHeight` is a **new, dedicated** constant ‚Äî do NOT grow the shared `switcherContentHeight` (196pt), which Home/Weather/default/NowPlaying all still rely on unchanged. Wire it through the same explicit-`height:`-override-wins mechanism `trayContentHeight` already established (Phase 44/D-08 precedent) at `blobShape`'s call site for `.calendarExpanded`.
- `calendarWidth`/`calendarContentHeight`/day-list row padding are this phase's three geometry call sites (mirrors Phase 44's "3-site geometry rule") ‚Äî verify all three stay in sync, same convention as `traySize`/`trayContentHeight`.
- D-07's forced-trailing popover direction must be re-verified on-device once the button moves to the day-list column's left edge ‚Äî confirm it never visually overlaps or clips against the month-grid column, which is the exact failure mode CALVIEW-06 exists to fix in a new spot.
- 240pt popover width, 472pt `calendarWidth`, and 220pt `calendarContentHeight` are starting points for on-device tuning, per this project's established sizing-iteration convention (D-11, and Phases 18/20/21/23/25/26/33's own precedent) ‚Äî flag for an on-device pass during execution, not a hard gate.
- `NotchWindowController.handleQuickAdd(_ kind: QuickAddKind, title: String)` (line ~1651) signature must grow to accept the picked `start`/`end` (Event) or `dueDate` (Reminder) `Date`(s) ‚Äî `CalendarService.createEvent(title:start:end:completion:)` and `createReminder(title:dueDate:completion:)` already accept full `Date` parameters, no EventKit-layer changes needed.
