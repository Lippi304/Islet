## Preview Visual Contract (SC#1, SC#2)

- **Trigger:** hovering the collapsed idle pill (`IslandPresentation.idle`) when `nowPlaying.hasPlayedSinceLaunch == true` AND `nowPlaying.presentation == .none` (nothing currently live).
- **Gate (SC#2):** if `hasPlayedSinceLaunch == false`, hover on the idle pill renders NOTHING new — today's unchanged behavior. This is a hard precondition check, not a visual state — no "disabled preview" look exists.
- **Rendered content on hover:** `mediaWingsRow(.playing(title: lastKnownTrack!.title, artist: lastKnownTrack!.artist), art: lastKnownTrack!.artwork)` — pixel-identical to the live Now Playing collapsed glance (same `Self.wingsSize`, same paddings, same shape/morph identity via `mediaWingsOrToast`'s existing `NotchShape`/`matchedGeometryEffect` wrapper).
- **Why `.playing` not `.paused`:** D-02 locks that bars animate exactly as the live-playing state — `isPlayingFor(.paused(...))` returns `false` and freezes the bars, which is NOT what D-02 asks for. Constructing `.playing(...)` (a presentation state that doesn't reflect "currently live playback" literally, only "bars should bounce") is the correct synthetic value for this rendering path — this is a display-only construction, never fed back into `nowPlaying.presentation` itself.
- **No frozen/faded variant:** do not build a dimmed or desaturated version of the wings row for "preview" vs "live" — the two must be visually indistinguishable per SC#1's "same visual as the active Now Playing view."

## Interaction Contract (D-01, D-04)

| Interaction | Behavior |
|---|---|
| Hover in | Idle pill morphs to the wings-preview shape (reuses `mediaWingsOrToast`'s existing shape/matchedGeometryEffect machinery — same morph the live media-wings state already performs from `.idle`). |
| Click (while preview showing) | Calls `nowPlayingMonitor?.togglePlayPause()` directly (same call site as line 1704). Per D-01, the pill stays exactly as the wings-preview shape — no expansion to the full Home transport view. |
| Hover out | Preview collapses back to `.idle` after the SAME ~0.4s `graceDelay` pointer-away timer already used everywhere else (`NotchWindowController.swift:401`) — no new timing constant, no new grace-period value for this specific case. |
| Resume fails (D-03) | See Resume-Failure Feedback below. |

## Resume-Failure Feedback Contract (D-03, SC#4)

Triggered when `togglePlayPause()` does not actually resume playback for a fully-stopped session (the open technical question flagged in 53-CONTEXT.md — verify feasibility early in planning; this is the contract for WHEN it fails, not a statement that it always will).

- **Placement:** replaces the `EqualizerBars` slot only (right wing) — the album art on the left stays visible, the wings row shape/size does not change.
- **Copy:** **"Wiedergabe nicht möglich"** — mirrors the exact grammatical pattern of the existing "Now Playing nicht verfügbar" / "Wetter nicht verfügbar" health-state texts (German, "nicht + adjective" construction), rather than the English "Can't resume" example phrase from 53-CONTEXT.md's D-03 (that context note describes the SHAPE of the feedback, not literal shipped copy — this app's established health-state convention is German; see `mediaUnavailableContent` and `weatherUnavailableContent`). Confirm/override if a different string is preferred.
- **Typography:** 11pt, medium weight, `design: .rounded`, `Color.white` — scaled down from the 14pt expanded-blob precedent to fit the 32pt-tall wings row (matches `toastTextRow`'s 12pt precedent for fitting compact text inside this same wings geometry; 11pt chosen because the failure string is longer than "Title — Artist" and must fit the ~90-110pt-wide right-wing slot without wrapping).
- **Layout:** single-line `Text`, `.lineLimit(1)`, right-aligned within the same trailing padding (`24pt`) the `EqualizerBars` view uses today — no new alignment rule.
- **Timing:** shown in place for the same ~0.4s `graceDelay` window as a normal hover-out, then the preview collapses to idle exactly as it would after a successful resume click or a hover-away (D-03: "then the preview collapses" — reuses the existing collapse mechanism, does not invent a new dismiss timer).
- **No new visual language:** no toast, no shake, no color change (no red/error tint) — text-only, matching this app's existing restrained health-state precedent.
