<objective>
Build the onboarding-replay mechanism (D-07/D-08/D-12): a new, narrow
`replayOnboarding()` entry and `finishOnboardingReplay()` exit on
`NotchWindowController` that reuse Phase 26's existing carousel verbatim but never touch
persisted onboarding state and correctly restore whatever the island was showing before

no cancel affordance today. Also adds `requestBluetoothPermission()`, the cross-window

since CoreBluetooth has no standalone request-authorization call to mirror.

device wing active, etc.) without clobbering island state or persisted flags — the one
genuinely tricky part of this phase per RESEARCH.md.
