    Build and run Islet in Debug from Xcode (Cmd-R), then on-device:

    1. **Picker-vs-Tray size match (DRAG-02)** — drag a real file from Finder toward the notch
       until the Quick Action picker (Drop/AirDrop/Mail) appears. Note its visible box size. Then
       drop the file (or discard the drag and instead switch to the Tray tab with a file already

       helpful). Expected: the picker's box is now visibly the same width AND height as the real
       Tray view — no more visibly-smaller-box mismatch.

    2. **Click-through hover→expand→move-down trace (D-08a, CR-01/CR-02-class regression check)**

       island, normal click-through to whatever is behind the island in both cases.

    3. **Button tap-zone re-check (D-08b)** — during an active file drag with the picker showing,
       move the dragged file over each of the 3 buttons (Drop, AirDrop, Mail) in turn and confirm

       confirm hover-highlight lands correctly on all 3) to confirm the tap zones track the bigger
       card correctly, not just visually but interactively.

    4. **TRAY-06 re-verification** — switch to the Tray tab with a typical number of files staged
       (e.g. 5-10 files) and confirm every file icon still fits without visual squeeze or overlap,
