<objective>
Fix DRAG-02: the Quick Action picker (the 3 Drop/AirDrop/Mail buttons shown during an in-progress
file drag) currently renders in a box hardcoded to `expandedSize.width` (420pt) and
`quickActionPickerContentHeight` (117pt) — visibly smaller than the real, already-widened Tray
view (`NotchPillView.traySize.width` = 650pt, `trayContentHeight + switcherRowHeight` = 189pt),
so the 3 buttons look cramped/misplaced relative to the landed-state Tray box the user compares
them against. TRAY-06 (the Tray's own squeeze) has no active reproduction per D-01 — this plan
does not touch Tray's own dimensions, only the picker's.

Purpose: Make the picker's card exactly match the real Tray footprint (width AND height, per
D-03's "genauso breit und tief" framing, which supersedes the ROADMAP's narrower width-only
wording) by reusing Phase 32's existing `traySize`/`trayContentHeight` constants at all three
geometry sites this codebase's own "three-site rule" requires — never inventing new numbers.

Output: `quickActionPickerFrame` (panel reservation), the `.quickActionPicker` `contentSize`
branch (click-through hot-zone), and `quickActionPickerView()`'s `blobShape` call all agree
pixel-for-pixel on the new Tray-aligned width/height; the now-unused `quickActionPickerContentHeight`
constant is removed; a lock-in unit test confirms `computeQuickActionButtonFrames` still produces
valid in-bounds button frames at the new dimensions.
</objective>

**Critical gotcha:** `trayFullView` passes `showSwitcher: true`, so `blobShape`'s own internal
`totalHeight` computation (`baseHeight + (showSwitcher ? Self.switcherRowHeight : 0) + ...`)
auto-adds `switcherRowHeight` on top of `trayContentHeight` for it — that is why `trayContentHeight`
alone (145) is sufficient there. `quickActionPickerView()` keeps `showSwitcher: false` (D-06), so
`blobShape` will NOT auto-add that addend for the picker. The picker's `height:` argument must
therefore be the explicit sum `Self.trayContentHeight + Self.switcherRowHeight`, not a bare copy
of `trayContentHeight`. Getting this wrong silently produces a picker shorter than the real Tray
despite "using the same height logic."

```swift
func computeQuickActionButtonFrames(card: CGRect) -> [CGRect] {
    let horizontalInset: CGFloat = 16
    let buttonRowHeight: CGFloat = 59
    let bottomInset: CGFloat = 16
    let gap: CGFloat = 16
    let rowRect = CGRect(x: card.minX + horizontalInset, y: card.minY + bottomInset,
                          width: card.width - 2 * horizontalInset, height: buttonRowHeight)
    let colWidth = (rowRect.width - 2 * gap) / 3
    return (0..<3).map { i in
        CGRect(x: rowRect.minX + CGFloat(i) * (colWidth + gap), y: rowRect.minY,
               width: colWidth, height: rowRect.height)
    }
}
```
