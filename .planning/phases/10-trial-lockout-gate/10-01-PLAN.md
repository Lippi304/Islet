<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App process ↔ Keychain | OS-trusted local store; still tamperable by a local user/attacker via Keychain Access.app or `security` CLI, but not by `defaults delete`/reinstall alone |
| App process ↔ UserDefaults/plist | Freely editable via `defaults write <bundle-id>` by anyone with local shell access — must never be sole authority for trial state |
| DEBUG build ↔ Release build | Compile-time boundary; DEBUG-only override code must never execute (or even exist) in a shipped Release binary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-01 | Tampering | `TrialManager` (Keychain trial-start) | mitigate | Keychain-backed start date (`kSecClassGenericPassword`) is queried independently of the UserDefaults mirror; `defaults delete`/reinstall does not clear it (D-10) |
| T-10-02 | Tampering | `TrialManager.trialStartDate()` reconciliation | mitigate | Earliest-of-two-known-dates wins for enforcement — editing only the UserDefaults mirror to a later date cannot extend the trial (Pitfall 5) |
| T-10-03 | Elevation of Privilege | `LicenseState.status` DEBUG override read path | mitigate | The override constant/enum AND the read-site inside `status` are both compiled only under `#if DEBUG`; a stray leftover UserDefaults key cannot silently unlock a Release build because the reading code doesn't exist there (Pitfall 4) — verified via a Release-configuration build in Task 3 |
</threat_model>
