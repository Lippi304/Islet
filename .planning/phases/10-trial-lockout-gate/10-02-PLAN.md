<objective>
Wire `LicenseState.shared.isEntitled` into the app's single visibility arbiter as a new dominant AND-term, add a best-effort one-shot expiry timer mirroring the controller's existing four-timer idiom, and add an idle-state guard so a license-state-driven hide never abruptly yanks the island out from under an active hover/expansion (D-13).

Purpose: LIC-03 requires a hard lockout — no pill, no activities, no expansion — the instant the trial expires (or the stub license is forced invalid), enforced in exactly the one place the codebase already funnels every show/hide decision through (`shouldShow(...)` inside `updateVisibility()`), per the established Pattern 7 discipline from Phases 2/6/8/9. D-13 additionally requires that this enforcement never abruptly interrupts an in-progress hover/expansion — it must defer to the next natural UI transition.
Output: `shouldShow(...)` gains an `isLicensed: Bool` parameter (breaking signature change, all call sites updated); `NotchWindowController` reads the live license state on every visibility check, schedules one proactive expiry-check timer, and defers a license-driven hide via a `pendingLockoutHide` flag until the pointer is no longer in the hot-zone and the island is no longer expanded.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `shouldShow(...)` call site ↔ rest of the codebase | This is the app's ENTIRE local feature-entitlement access-control surface (ASVS V4, partial) — a second show/hide call site anywhere else would silently bypass it |
| Dispatch timer ↔ wall clock | `DispatchQueue.main.asyncAfter` deadlines are Mach-time-based and pause during system sleep — a multi-day timer can fire late relative to wall-clock expiry |
| License-state transition ↔ active pointer interaction | A license state change (trial expiry, DEBUG stub flip) can occur at any instant, including while the pointer is mid-hover or the island is mid-expansion — the enforcement must not abruptly interrupt that interaction (D-13) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04 | Elevation of Privilege | `FullscreenDetector.shouldShow(...)` isLicensed AND-term | mitigate | `isLicensed` is a dominant AND-term inside the single visibility arbiter; the ONE call site in `updateVisibility()` is the only place it is wired — no second show/hide call site exists anywhere in the file that could bypass it (Pattern 7 / D-11) |
| T-10-06 | Tampering (self-limiting) | `NotchWindowController.trialExpiryWorkItem` across system sleep | accept | Mach-time deadlines only ever fire LATE (never early) across sleep, so the wall-clock `licenseState.isEntitled` check inside `updateVisibility()` (re-run on every existing screen/space/app-activate trigger) remains authoritative; worst case is a few extra hours of trial — an accepted, bounded cost (Pitfall 1, Assumption A4, D-12) |
| T-10-07 | (UX-adjacent, not classic STRIDE) | Mid-interaction abrupt lockout (D-13) | mitigate | `updateVisibility()`'s idle-state guard (`pendingLockoutHide`) defers a license-driven hide whenever `pointerInZone \|\| interaction.isExpanded` is true, and the deferred hide is re-applied deterministically at the next natural transition (`handleHoverExit`'s grace-elapsed collapse, or a toggle-shut click in `handleClick`) — this is an engineered guard, not just an incidental side-effect of existing call timing; on-device confirmation remains in Plan 10-04's manual checkpoint |
