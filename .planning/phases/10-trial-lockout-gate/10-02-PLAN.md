<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `shouldShow(...)` call site ↔ rest of the codebase | This is the app's ENTIRE local feature-entitlement access-control surface (ASVS V4, partial) — a second show/hide call site anywhere else would silently bypass it |
| Dispatch timer ↔ wall clock | `DispatchQueue.main.asyncAfter` deadlines are Mach-time-based and pause during system sleep — a multi-day timer can fire late relative to wall-clock expiry |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-04 | Elevation of Privilege | `FullscreenDetector.shouldShow(...)` isLicensed AND-term | mitigate | `isLicensed` is a dominant AND-term inside the single visibility arbiter; the ONE call site in `updateVisibility()` is the only place it is wired — no second show/hide call site exists anywhere in the file that could bypass it (Pattern 7 / D-11) |
| T-10-06 | Tampering (self-limiting) | `NotchWindowController.trialExpiryWorkItem` across system sleep | accept | Mach-time deadlines only ever fire LATE (never early) across sleep, so the wall-clock `licenseState.isEntitled` check inside `updateVisibility()` (re-run on every existing screen/space/app-activate trigger) remains authoritative; worst case is a few extra hours of trial — an accepted, bounded cost (Pitfall 1, Assumption A4) |
