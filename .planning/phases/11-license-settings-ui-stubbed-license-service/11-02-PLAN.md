<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Settings `TextField` → LicenseService | Untrusted user-typed license key crosses into the (fake) validator |
| activation success → UserDefaults trigger → NotchWindowController arbiter | The unlock signal crosses from the view to the single visibility arbiter |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-02 | Elevation of Privilege | `sessionActivated` / `license.activationNudge` UserDefaults key | mitigate | The written UserDefaults key is a TRIGGER ONLY, never read as entitlement truth; entitlement lives in the in-memory `sessionActivated`, which resets to `false` on relaunch. A leftover key grants nothing on next launch (verify: activate → quit → relaunch → app back in trial/expired). |
| T-11-03 | Tampering | `enteredKey` TextField input | mitigate | Key is passed only to `licenseService.activate` (opaque `==` compare downstream) and to the empty-check `trimmingCharacters`; never interpolated into a URL, shell, or log sink. |
| T-11-04 | Tampering | Buy Now URL | mitigate | URL is a hardcoded constant `https://getislet.app` opened via `NSWorkspace.shared.open`; no user input flows into the URL, so no injection surface. |
| T-11-01 | Elevation of Privilege | reachable magic-key path (shared with Plan 01) | mitigate | Handled at the service (Plan 01): Phase 12 replaces the stub before Phase 13 distribution. |
| T-11-SC | Tampering | package installs | accept | N/A — no package installs; UI-only change to an existing Swift file. |
</threat_model>
