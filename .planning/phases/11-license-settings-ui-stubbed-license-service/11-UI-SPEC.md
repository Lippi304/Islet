# Phase 11 — UI Design Contract

> Visual and interaction contract for the License section of the Settings window.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Platform note:** This is a native macOS SwiftUI app, not a web/React project. The
> shadcn initialization gate is **not applicable** — there is no `components.json`,
> Tailwind, or component registry. The template's web idioms (px, hex, shadcn blocks)
> are mapped to their native SwiftUI equivalents below: SwiftUI semantic font styles,
> `Color` semantic roles, SF Symbols, and the existing `Form`/`Section`/`LabeledContent`
> layout already established in `SettingsView.swift`. The contract stays prescriptive by
> naming the exact SwiftUI control, modifier, and copy for each element.

## Design System

| Property | Value |
|----------|-------|
| Tool | none (native SwiftUI — no shadcn/Tailwind/registry) |
| Preset | not applicable |
| Component library | SwiftUI native controls (`Form`, `Section`, `LabeledContent`, `TextField`, `Button`, `Text`) |
| Icon library | SF Symbols (system) + inline glyphs already in use (`✓`, `✗`, `⟳`) |
| Font | System (San Francisco) via SwiftUI semantic text styles — never hardcoded point sizes |

**Existing surface this phase extends:** `Islet/SettingsView.swift` — a single `Form`
at `.frame(width: 360)` with `.padding(20)`, using the `Section(...)` / `LabeledContent`
idiom and `.appearsActive` refocus re-sync. The new License section slots in at the
**top of this Form**, above "Launch Islet at login" (CONTEXT D-02).

## Spacing Scale

Native SwiftUI spacing. The existing `SettingsView` already establishes two
non-8-point conventions that this phase **must reuse for visual consistency** (do not
re-specify them away):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Tight inline gaps (glyph → text in the status line) |
| sm | 8pt | Vertical gap between the license field, Activate button, and status line |
| md | 12pt | Gap between the status line and the days-remaining / expired message |
| lg | 16pt | Intra-section vertical rhythm inside the License `Section` |

**Exceptions (established codebase conventions — inherited, not introduced by this phase):**
- Form outer padding: `.padding(20)` — existing, keep.
- Form fixed width: `.frame(width: 360)` — existing, keep. The License section inherits
  this width; the license `TextField` should fill available width (`.frame(maxWidth: .infinity)`).
- Swatch row `HStack(spacing: 10)` — existing Accent row, not touched by this phase.

Prefer SwiftUI's default `Section`/`Form` intrinsic spacing over hand-tuned paddings;
only add explicit spacing (from the table above) where the default rhythm reads cramped
(e.g. the inline status line directly under the field).

## Typography

Declared via **SwiftUI semantic text styles** (they scale with the system and Dynamic
Type — never hardcode px). Exactly 3 roles are used in this section, plus one existing
inherited role:

| Role | SwiftUI style | Weight | Usage |
|------|---------------|--------|-------|
| Primary message | `.headline` | semibold (style default) | The expired call-to-action ("3-day trial period expired") — the one prominent line when the island is locked (CONTEXT D-01) |
| Body / status | `.body` | regular (style default) | Days-remaining countdown line; "Licensed ✓" confirmation; inline Activate status line |
| Caption | `.footnote` | regular (style default) | Secondary helper text if any (matches the existing trial-notice line being replaced) |

Weights: exactly **2** — regular (body/footnote defaults) and semibold (`.headline`
default). Do not introduce custom `.fontWeight(...)` beyond these. Line height is the
SwiftUI style default for each role (do not override).

## Color

Native macOS semantic colors — they auto-adapt to light/dark mode and accent settings.
The 60/30/10 split is expressed in SwiftUI semantic terms:

| Role | SwiftUI value | Usage |
|------|---------------|-------|
| Dominant (60%) | window/`Form` background (system default) | Settings window surface — untouched |
| Secondary (30%) | `.foregroundStyle(.secondary)` | Days-remaining countdown, helper/caption text, de-emphasized labels |
| Accent (10%) | `.green` (system) | **Success only:** the `✓ License activated` status line |
| Destructive | `.red` (system) | **Failure only:** the `✗ That key wasn't recognized.` status line |

**Accent reserved for:** the successful-activation confirmation glyph + text
(`✓ License activated`) and the "Licensed ✓" checkmark. It is **not** used to tint the
Buy Now or Activate buttons — those use the standard system button styling (`.borderedProminent`
for the primary Activate/Buy action is acceptable, which already carries the system accent;
do not add a second custom accent color).

Notes:
- The `⟳ Validating…` transient state uses `.secondary` (neutral, not colored) — color is
  reserved for the terminal success/failure outcome only.
- Do NOT reuse the Activity **accent palette** (`ActivitySettings.palette`: white/blue/green/
  orange/pink/purple) here — that palette is the island-tint preference, a separate concern.
- Idle state shows **no status line and no color** (CONTEXT D-04).

## Copywriting Contract

All copy is locked by CONTEXT.md decisions D-01, D-03, D-04, D-07 and §Specific Ideas.
Tone: neutral and plain — no scary error dialogs (CONTEXT §Specifics).

| Element | Copy |
|---------|------|
| Days-remaining line (`.trial`) | `{n} days left in your trial.` — singular for 1: `1 day left in your trial.` (value from `LicenseState.status → .trial(daysRemaining:)`, already rounds up / clamps to min 1) |
| Expired heading (`.trialExpired`) | `3-day trial period expired` — rendered `.headline`, the prominent primary CTA when locked |
| Licensed confirmation (`.licensed`) | `Licensed ✓` — Buy Now + key field hidden in this state (CONTEXT D-01) |
| Primary CTA — Buy Now | `Buy Islet — €7.99` — opens `https://getislet.app` in default browser (placeholder; real Polar URL is Phase 12). Hidden in `.licensed` (CONTEXT D-07) |
| Primary CTA — Activate | `Activate` — disabled while validating; disabled when the field is empty (CONTEXT D-04/D-05) |
| License field placeholder | `Enter your license key` (`TextField` prompt) |
| Status — validating | `⟳ Validating…` (`.secondary`; shown ~1s simulated round-trip, CONTEXT D-06) |
| Status — success | `✓ License activated` (`.green`) |
| Error state | `✗ That key wasn't recognized.` (`.red`) — the only failure copy; no alert/dialog |
| Empty state | Not applicable — the section always renders one of the three `LicenseStatus` states; there is no "no data" empty state. Empty *field* input is inert (no validation attempt), not an empty state. |
| Destructive confirmation | None — this phase has no destructive actions (no revoke/deactivate/reset in-UI). No confirmation dialog required. |

**Interaction state machine (CONTEXT D-04/D-05/D-06 — locked):**
`idle` (no status line) → tap Activate → `⟳ Validating…` (~1s, Activate disabled) →
- valid magic key (`ISLET-DEMO-OK`, DEBUG-documented stub literal) → `✓ License activated`
  → flips entitled state in-memory for the session → funnels through Phase 10's
  `NotchWindowController.updateVisibility()` so a locked island reappears at the next
  natural UI transition (no abrupt yank — Phase 10 SC#5 / Pitfall 5).
- any other non-empty input → `✗ That key wasn't recognized.` (returns to idle-able state,
  field editable, Activate re-enabled).
- empty input → no state change, no network-ish attempt.

**Adaptive section rule (CONTEXT D-01):** one `Section` whose body swaps on
`LicenseState.status`:
- `.trial(daysRemaining:)` → days-remaining line + Buy Now + license field/Activate.
- `.trialExpired` → expired `.headline` message + Buy Now + license field/Activate.
- `.licensed` → `Licensed ✓` only (Buy Now + field hidden).
