final class BluetoothMonitor: NSObject {
    private nonisolated(unsafe) var connectToken: IOBluetoothUserNotification?
    private nonisolated(unsafe) var running = false

    func start() {

    }

    nonisolated func stop() {
        connectToken?.unregister()
        connectToken = nil
        running = false
    }
}

private func defaultOutputDeviceID() -> AudioDeviceID? {
    var deviceID = AudioDeviceID(0)
    var deviceIDSize = UInt32(MemoryLayout<AudioDeviceID>.size)
    var outputAddr = AudioObjectPropertyAddress(
        mSelector: kAudioHardwarePropertyDefaultOutputDevice,
        mScope: kAudioObjectPropertyScopeGlobal,
        mElement: kAudioObjectPropertyElementMain)
    guard AudioObjectGetPropertyData(AudioObjectID(kAudioObjectSystemObject), &outputAddr, 0, nil, &deviceIDSize, &deviceID) == noErr
    else { return nil }
    return deviceID
}

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| CoreAudio system HAL (local, public API) | Device UID/name/channel-count/volume-support facts come from the local machine's own audio hardware abstraction layer — not network, not user-typed input, not another process's untrusted data. |
| System audio-output state (write) | `setDefaultOutput`/`hasVolumeControl` mutate/query real system audio routing — a local, single-user, no-auth operation, same trust tier as the existing `adjustSystemVolume()`/`toggleSystemMute()` in `VolumeReader.swift`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-47-03 | Tampering | AudioOutputMonitor.setDefaultOutput | mitigate | Re-resolves UID→AudioDeviceID immediately before every write (never a stale cached ID, Pitfall 4), guards every AudioObjectSetPropertyData/GetPropertyData call against non-noErr, and confirms the switch actually stuck via a delayed re-read (Pitfall 8) before ever reporting success. |
| T-47-04 | Spoofing | Device UID/name CFStrings read from CoreAudio | accept | Values originate from the local system's own audio HAL (not network/untrusted-user input); passed as plain String data only, never interpolated into a shell command or format string — mirrors BluetoothMonitor.device.name's existing "untrusted but inert" discipline. |
| T-47-05 | Denial of Service | AudioObjectAddPropertyListenerBlock registration | mitigate | Idempotent start() (running guard) prevents duplicate listener registration/callback storms on repeated start() calls; full teardown in stop() removes both registered listeners so no callback can outlive the monitor's owner. |

No Repudiation / Elevation-of-Privilege entries — this plan introduces no identity/audit boundary
and requests no new system privilege beyond the app's existing (unsandboxed) entitlement set.
</threat_model>
