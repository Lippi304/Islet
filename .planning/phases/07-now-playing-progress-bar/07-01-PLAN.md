<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Third-party media player -> mediaremote-adapter -> NowPlayingMonitor | Untrusted numeric metadata (duration/elapsed/timestamp/rate) originates from whatever app is "Now Playing" (Spotify/Apple Music today, but the payload shape is attacker-influenceable by any app that can become the system's Now Playing source) |
| NowPlayingMonitor -> pure seam (NowPlayingPresentation.swift) | The 4 raw Double? fields cross from thin glue into pure mapping/formula functions |
| Pure seam -> NotchPillView (ProgressBar) | Computed elapsed/duration values cross into SwiftUI rendering (bar width, label text) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | `playbackPosition(from:)` (NowPlayingPresentation.swift) | mitigate | All 4 new fields are `Double?`; a malformed/garbage/negative value from a misbehaving player still produces a well-typed `PlaybackPosition` or `nil` — never a crash. No force-unwraps anywhere in the mapping. |
| T-07-02 | Denial of Service | `ProgressBar` fraction computation (NotchPillView.swift) | mitigate | `fraction = total > 0 ? min(max(elapsed / total, 0), 1) : 0` — a zero/negative duration or an elapsed value outside `[0, total]` (e.g. a player reporting elapsed > duration) can never produce a NaN width or an overflowing `Capsule` frame; Task 2 acceptance criteria enforce this clamp exists. |
| T-07-03 | Information Disclosure | Playback position/duration values rendered in the UI | accept | This data already flows through the existing Phase-4 MediaRemote pipeline (title/artist/artwork); elapsed/duration timing is not sensitive (no PII, no credentials, visible on-screen by design per PBAR-01). No new exposure surface introduced. |
| T-07-04 | Elevation of Privilege | ProgressBar interaction surface | mitigate | The bar has zero `.onTapGesture`/`.gesture`/`Button` attached (Task 2 acceptance criteria grep-enforced) — it cannot be used to trigger playback control or any other action beyond passive display, per PBAR-01's "no tap-to-seek" requirement. |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | Not applicable — this phase introduces zero new package installs (07-RESEARCH.md Package Legitimacy Audit: "Not applicable"). The only external dependency touched, MediaRemoteAdapter, is the existing Phase-4 pin, unchanged and independently re-verified in 07-RESEARCH.md. |
</threat_model>
