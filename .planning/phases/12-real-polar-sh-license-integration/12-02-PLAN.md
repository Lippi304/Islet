<objective>
Create `PolarLicenseService`: the real Polar.sh customer-portal license-key validation client,
a ZERO-protocol-change drop-in for the existing `LicenseService` protocol. It POSTs the pasted
key to Polar over `URLSession`, maps the HTTP outcome to `.success` / `.invalidKey` /
`.unreachable(String)` with the strict taxonomy from research, and always hops completion back
to the main thread. The network is behind an injectable `HTTPSession` seam so the verdict logic
is unit-testable without hitting `api.polar.sh`.

Purpose: this is the LIC-02 online-validation half. The critical safety property (D-04): a
transient failure (5xx / URLError) maps to `.unreachable` (retryable), NEVER `.invalidKey` â€” a
paid key is never falsely rejected on a network blip.
Output: `Islet/Licensing/PolarLicenseService.swift`, `IsletTests/PolarLicenseServiceTests.swift`.
This plan runs in parallel with 12-01 (no shared files); the service is PURE (returns a verdict,
does not persist) so it has no dependency on the Keychain store.
</objective>
