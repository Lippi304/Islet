# Phase 30 — UI Design Contract

> Visual and interaction contract for Home Music-Only. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Platform note:** This is a native macOS Swift/SwiftUI app (`NotchPillView.swift`), not a web/React project. The shadcn gate and CSS-token conventions in the template don't apply — there is no `components.json`/Tailwind config to detect. All values below are the project's own established SwiftUI conventions (SF Symbols, `.rounded` font design, the Phase 25 gradient material), extended for this phase's 3 new/changed elements only:

1. Last-played state (reuses `mediaExpanded` exactly — D-04)
2. Transport button hover background (new — D-05)
3. Home empty state (new — D-09/D-10)

## Layout & Interaction Contract

*(Supplementary section — this phase's actual design surface is 3 view-states routed by `IslandResolver`, not a token table alone.)*

### State 1 — Playing & State 2 — Last-played (HOME-01 / HOME-02, D-01/D-02/D-04)

Both states render through the **same** `mediaExpanded(_:art:)` view function — no new view, no branch that hides controls. The only difference between the two states is the data fed in:
- **Playing**: `nowPlaying.presentation` (`.playing`/`.paused`) + `nowPlaying.artwork`, live.
- **Last-played**: the new `NowPlayingState.lastKnownTrack` sticky field (title/artist/artwork; session-only, D-07/D-08) fed into the identical layout.

Existing layout (unchanged): 40×40pt rounded-square album art (or `music.note` placeholder) top-left, title (15pt bold) + artist (12pt secondary) stacked to its right, `EqualizerBars` top-right (frozen/static when not actually playing — existing `isPlayingFor` gate, D-04/D-05 in `NowPlayingMonitor` conventions, not this phase's D-04), `ProgressBar` below, then the centered transport row (⏪ ⏯ ⏩) with reserved Shuffle/Repeat slots.

**New in this phase (D-05):** each of the 3 real transport buttons (`transportButton(_:action:)`) gets a hover-triggered rounded-rectangle background:

| Property | Default value | Status |
|----------|---------------|--------|
| Shape | `RoundedRectangle(cornerRadius: 8, style: .continuous)` | default, tune on-device |
| Size | 32×32pt (4pt of breathing room around the existing 28×28 icon frame) | default, tune on-device |
| Fill | `Color.white.opacity(0.12)` on hover, `Color.clear` otherwise | default, tune on-device |
| Trigger | Native `.onHover` (this is a mouse-driven desktop app; per-button local hover state, not a global gesture) | locked mechanism, exact fill/size tunable |
| Scope | Only the 3 real buttons (prev/playpause/next). The reserved Shuffle/Repeat `Color.clear` placeholder slots (28×28, D-09 "not built") get **no** hover box — they are not interactive. | locked |
| Animation | Wrap the opacity/fill change in the existing `.easeOut` / short-duration transition already used for hover affordances elsewhere in this file (e.g. collapsed-pill hover bounce) — no new animation curve invented for this one element. | default |

Per 30-CONTEXT.md: exact corner radius/size/opacity are explicitly Claude's Discretion, expected to be refined during on-device UAT (this project's established pattern for micro-visual tuning — do not treat the values above as final, treat them as the starting point the executor implements first).

### State 3 — Empty (HOME-03, D-03, D-09, D-10)

New `homeEmptyState` view, structurally identical to `trayEmptyState` (`NotchPillView.swift:745-760`) — same `VStack(spacing: 4)`, same `.padding(.top, 24)`, same `.frame(maxWidth: .infinity, maxHeight: .infinity, alignment: .top)` outer frame, same icon→heading→body ordering:

```swift
private var homeEmptyState: some View {
    VStack(spacing: 4) {
        Image(systemName: "music.note")
            .font(.system(size: 28))
            .foregroundStyle(.white.opacity(0.4))
        Text("Nothing Playing")
            .font(.system(size: 13, weight: .semibold, design: .rounded))
            .foregroundStyle(.white)
        Text("Start something in Spotify or Music.")
            .font(.system(size: 11, weight: .regular, design: .rounded))
            .foregroundStyle(.secondary)
            .multilineTextAlignment(.center)
    }
    .padding(.top, 24)
    .frame(maxWidth: .infinity, maxHeight: .infinity, alignment: .top)
}
```

Icon: SF Symbol `music.note` (D-09 leaves the exact symbol to discretion — chosen because it's the *same* glyph already used by `artThumbnail`'s nil-artwork placeholder, keeping the icon vocabulary consistent within this one feature rather than introducing a new symbol like `music.note.list` or `waveform`).

This view is wrapped in the same `blobShape(topCornerRadius: 24, bottomCornerRadius: 32, alignment: .top, showSwitcher: true) { ... }` call shape every other Home/switcher-row presentation uses (`expandedIsland`/`calendarFullView`/`mediaExpanded` precedent) — not a new panel size or shape.

### Removed: `expandedIdle` glance (HOME-03)

The weather/date/calendar `expandedIsland` HStack (lines 436-459) is removed from the Home path entirely. `IslandResolver.resolve()`'s `.expandedIdle` branch (currently returning when `nowPlaying == .none`, lines 102-104) is replaced by branching on `hasPlayedSinceLaunch`: `true` → last-played `mediaExpanded`, `false` → `homeEmptyState`. This is a resolver/routing change, not a new visual — confirm before deleting whether `IslandPresentation.expandedIdle` is referenced anywhere else (canonical_refs already flags this; Weather/Calendar have their own dedicated cases per Phase 28).
