`acceptsMouseMovedEvents = true`. Every existing hover interaction in this app (the notch
expand-on-hover itself) is driven by a manual global `NSEvent` monitor, NOT native window

`.onHover` in the codebase â€” without `acceptsMouseMovedEvents`, the window never receives
`mouseMoved`, so `.onHover` can never fire, regardless of button position.

Camera clearance: user confirmed on-device the overlap is minor; bump `NotchPillView.cameraClearance`
from 32pt to 37pt (+5pt) to add headroom for `mediaExpanded`'s art/title/transport content.
</objective>
