## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| CLI toolchain mismatch | See blocking constraint above | blocking | Always `export DEVELOPER_DIR=.../Xcode-beta.app/Contents/Developer` before any `xcodebuild` verification in this project |
| DerivedData wipe races Xcode GUI | See blocking constraint above | blocking | Re-run `-resolvePackageDependencies` immediately after any DerivedData wipe |
| Sparkle `file://` SUFeedURL | Sparkle (via `URLSession.dataTask`) cannot fetch `file://` URLs — fails with a generic "error retrieving update information" with no useful detail surfaced. | advisory | For any future local-appcast testing, serve the mock file over `http://127.0.0.1:PORT/...` (e.g. `python3 -m http.server`) instead of `file://`. Loopback HTTP needs no ATS exception. |
| Badge cross-case geometry via PreferenceKey | A GeometryReader + PreferenceKey + `.overlayPreferenceValue` + named coordinate space approach for anchoring the update badge to whichever collapsed-ish shape (idle/Charging/Device/Focus/OSD) is on screen did NOT work on-device — the badge rendered stuck near the container's origin regardless of active case, meaning the preference never propagated a real value. | advisory | Use `@State` + `onChange(of: proxy.size.width)` to track just the active shape's width, then `.overlay(alignment: .top)` with a width-matched `HStack`. Vertical alignment is free since `.top` matches `presentationSwitch`'s own position — no coordinate-space math needed. This is what NotchPillView.swift currently does and IS visually correct. |
