private func handleDevice(_ reading: DeviceReading) {
    let now = Date().timeIntervalSinceReferenceDate
    guard shouldShowDeviceSplash(address: reading.address, connected: reading.connected, now: now,
              lastShown: deviceLastShown, debounce: deviceDebounce, suppressedAtLaunch: deviceSuppressedAtLaunch)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| paired Bluetooth device → app | `handleDevice` receives an untrusted `DeviceReading` (device name) across the IOBluetooth boundary; it flows into the queue/resolver/view. |
| UserDefaults → controller | The controller reads the four settings keys to gate monitors + pick the accent — local, non-privilege, user-owned values. |
| MediaRemoteAdapter perl child → app | (Inherited, unchanged) Now-Playing data crosses from the spawned perl child; isolated in NowPlayingMonitor; this plan only toggles it on/off. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-09 | Denial of Service | `TransientQueue` flooding from a flapping device or rapid charging ticks reaching `handleDevice`/`handlePower` | mitigate | `shouldShowDeviceSplash(...)` debounce + at-launch burst suppression gate BEFORE the queue; the queue is itself bounded + de-duped (Plan 01). Both gates are active in `handleDevice`. Charging % ticks update the standing head, not enqueue. |
| T-06-10 | Tampering / Info-disclosure | Untrusted device name routed through the resolver into `deviceWings` | mitigate | Name already clamped to a plain bounded String by `deviceLabel(...)` (T-05-01) before becoming a `.device` transient; the view's `Text` is inert + `.lineLimit(1)/.truncationMode(.tail)`. No format/shell use. |
| T-06-11 | Tampering | Out-of-range persisted `accentIndex` injected into the hosting view | mitigate | `ActivitySettings.accent(for:)` clamps to the neutral default — a corrupted index can never crash the render. |
| T-06-12 | Denial of Service (resource leak) | IOBluetooth tokens / perl child leaking on toggle-off or controller death | mitigate | Toggle-off calls `bluetoothMonitor.stop()` / `nowPlayingMonitor.stop()` (full teardown); `deinit` calls both plus `powerMonitor.stop()` and cancels all work items; the UserDefaults observer is removed in deinit. |
| T-06-13 | Tampering | A second show/hide site racing the fullscreen/clamshell observers | mitigate | The resolver only decides WHAT renders; `updateVisibility()` stays the SOLE show/hide site (acceptance criterion: exactly one definition). No second `orderOut`/`orderFront` path is added. |

No threat reaches **high** severity: untrusted input is bounded and inert, resources are fully torn down, persisted values are clamped, and the single show/hide site is preserved. The security gate passes.
