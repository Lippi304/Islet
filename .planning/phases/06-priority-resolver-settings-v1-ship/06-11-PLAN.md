<objective>
Close two remaining low-priority findings from the fresh multi-agent code review of the already-
shipped Phase 6 Now-Playing bridge — both are cleanup/architecture, not runtime bugs:

1. **`TrackSnapshot.hasArtwork` dead field** — populated at construction (`NowPlayingMonitor.swift`)
   but never read by any consumer (the pure `nowPlayingPresentation(from:)` classifier ignores it; the
   real artwork presence is tracked separately via the `NSImage?` that flows straight to the
   `@Published` model, never through this pure seam).
2. **`NowPlayingMonitor` stored as a concrete class, not behind a protocol** — `NotchWindowController`
   holds `private var nowPlayingMonitor: NowPlayingMonitor?`, which is a direct violation of this
   project's own `CLAUDE.md` mandate: *"isolate all now-playing code behind one Swift protocol/service
   so swapping the implementation is a one-file change."* This is the architectural insurance CLAUDE.md
   explicitly calls for against a future Apple MediaRemote break — currently there is no protocol
   seam to swap behind.

Purpose: Remove genuinely dead state and close a gap against this project's own locked architectural
mandate (CLAUDE.md), without changing `NowPlayingMonitor`'s runtime behavior, its constructor
signature, or any of its call sites' argument lists.
Output: `hasArtwork` deleted from `TrackSnapshot`; a new `NowPlayingService` protocol in
`NowPlayingMonitor.swift`; the controller's stored property re-typed against it.
</objective>
