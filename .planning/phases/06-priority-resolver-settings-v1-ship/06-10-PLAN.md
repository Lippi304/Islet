<objective>
Fix two "plausible, needs on-device confirmation" bugs from a fresh multi-agent code review, both in
the expanded/collapsed island view and its controller-driven state:

1. **Ancestor `.onTapGesture` over nested transport `Button`s** — the whole island `ZStack` in
   `NotchPillView.body` carries a single `.onTapGesture { onClick() }` that sits ABOVE the nested
   transport `Button`s inside `mediaExpanded`. SwiftUI's gesture resolution between an ancestor
   `TapGesture` and a descendant `Button` is not guaranteed, so tapping play/pause/skip could also
   trigger the container's collapse/expand toggle. Fixed structurally (not by relying on undocumented
   SwiftUI internals): the tap-to-toggle gesture is scoped OFF the button row entirely so there is no
   competing gesture recognizer over it, eliminating the ambiguity by construction.
2. **Unconditional artwork overwrite** — `nowPlayingState.artwork` is set to whatever the latest
   snapshot carries, including `nil`, with no fallback to retain previously-loaded art for the SAME
   track. Since album art can arrive a beat after metadata (a documented latency case), this causes a
   visible flicker back to the placeholder when a callback doesn't carry artwork for a track that's
   already showing real art.

Purpose: Close two real UX papercuts in the Now-Playing expanded view without touching the pure
`nowPlayingPresentation(from:)` classification seam's existing behavior or signature.
Output: A restructured tap-gesture scope in `NotchPillView.swift`; a new pure `isSameTrack(_:_:)`
helper in `NowPlayingPresentation.swift` with unit tests; artwork-retention logic in
`NotchWindowController.handleNowPlaying`.
</objective>
