<objective>
Remove dead state and consolidate duplicated code that a fresh multi-agent code review flagged in the
already-shipped Phase 6 resolver/view layer — none of these are behavior bugs, all are maintainability
risks the review correctly identified as "a future fix to one copy not mirrored to the others silently
diverges behavior":

1. **`DeviceActivityState` dead code** — constructed, written at 5 controller call sites (real per-
   mutation bookkeeping cost), but NEVER passed to `NotchPillView` and has zero observers anywhere.
   Pure dead weight, unlike `chargingState` (Finding 10 below, ALSO unread by the view but at least
   passed to it) or `nowPlayingState` (genuinely read for `.artwork`).
2. **Redundant `@ObservedObject var charging: ChargingActivityState`** in `NotchPillView` — the view
   body reads zero properties of it; the charging % the wings render comes directly from the
   `IslandPresentation.charging(ChargingActivity)` case payload the resolver already re-renders on
   every mutation (including the in-place % tick).
3. **Duplicated enqueue-render-dismiss triplet** — `handlePower` and `handleDevice` each hand-roll the
   identical "spring-wrap renderPresentation, call the sole updateVisibility(), (re)arm the shared
   dismiss" sequence.
4. **Duplicated wings-shape skeleton** — `wings(for:)`, `mediaWings(_:art:)`, and `deviceWings(for:)`
   repeat the identical `NotchShape → .fill → .matchedGeometryEffect → .frame → .overlay(HStack...)`
   skeleton, even though their size constants (`wingsSize`/`mediaWingsSize`/`deviceWingsSize`) are
   already numerically identical (290×32, the post-checkpoint "one uniform width" decision).

Purpose: Reduce duplication risk and delete dead state before it accumulates further bookkeeping cost,
following the pattern-mapper's "pure-seam-first" and "ONE place" discipline already established
throughout this codebase (mirrors how `updateVisibility()` and `scheduleActivityDismiss()` are already
each a single call site).
Output: `DeviceActivityState.swift` deleted; a new `presentTransientChange()` helper in the controller;
a new `wingsShape(content:)` helper in the view; the redundant `charging` view property removed.
</objective>
