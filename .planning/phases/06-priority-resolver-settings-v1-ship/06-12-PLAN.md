<objective>
Fix two confirmed bugs in the Phase-0 release pipeline (`scripts/release.sh`) surfaced by a fresh code
review — both are real gaps relevant to Phase 6's own goal ("the app ships as a production notarized
release," ROADMAP Success Criterion 3), even though the real Developer-ID run itself remains a deferred
D-15 carry-over (no Apple Developer account yet):

1. **Only the DMG gets stapled, never the `.app` inside it.** `stapler` does not recurse into disk
   images. A user who drags `Islet.app` out of the DMG to `/Applications` gets a copy with NO local
   notarization ticket — first launch while offline (no network to fetch the ticket from Apple) can be
   blocked or badly delayed by Gatekeeper. Apple's own notarization docs require stapling BOTH the DMG
   and the `.app` inside it for DMG distribution.
2. **The dry-run SKIP banner always claims "(ad-hoc signed, NOT notarized)"** even when `DEVELOPER_ID`
   IS filled and only `NOTARY_PROFILE` is missing — in that case the app was actually just signed with
   the REAL Developer-ID certificate (Step 3 already branches on this correctly), so the banner's
   description is wrong and misleading about what was actually produced.

Purpose: Make the release pipeline correct for BOTH the eventual real run (D-15 carry-over) and any
partial-credentials dry-run state, without changing the script's placeholder-gating structure, its
`hdiutil`/UDZO choice, or anything else in the Phase-0 pipeline that already runs unchanged per D-15.
Output: Two targeted fixes in `scripts/release.sh`.
</objective>
