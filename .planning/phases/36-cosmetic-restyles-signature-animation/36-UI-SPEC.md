# Phase 36 — UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Scope note:** Three independent, pure view-layer restyles, all rendering inside Phase 35's Liquid Glass material (`islandFill` / `liquidGlassEffectLayer`, unchanged) — zero resolver/monitor/data changes, per `36-CONTEXT.md`:
1. **HUD-01/HUD-02** — Bluetooth/AirPods and Charging collapsed wings (`wings(for:)`, `deviceWings(for:)`, `deviceTrailing` in `NotchPillView.swift`) get the Droppy-pill visual language (icon+label left, status indicator right).
2. **EQ-01** — `EqualizerBars` gets a new bar geometry + a snappier periodic-reroll-and-spring motion, replacing today's continuous per-bar sine wave.
3. **ONBOARD-04** — The onboarding Welcome step's "Meet Islet" heading (`onboardingWelcomeStep`) is replaced by a handwritten-signature stroke-reveal animation; the body copy below it is untouched.

All three reuse existing chrome/shape helpers verbatim (`wingsShape`, `blobShape`, `chipButton` conventions) — this spec defines only the NEW visual/motion values layered on top.

## HUD Restyle Contract (HUD-01, HUD-02)

**Structural constraint (D-01, locked — not a style choice):** Islet's collapsed wings MUST keep flanking the physical camera bump (left wing / gap / right wing) — this is a chrome reskin of the existing split-wing structure (`wingsShape`, `Self.wingsSize` = 290×32), not a rebuild into one continuous Droppy-style capsule. Do not attempt to merge the two wings.

**Focal point:** the right-wing status indicator (green ring / green battery / percentage) is the primary focal point of each wing when present — it is the "is this good/active" signal a glance needs first. The left icon+label is secondary context (what device/what state). This ordering is why the right-wing color rule below is fixed-green rather than theme-tinted: the focal element must read consistently regardless of the user's chosen accent.

### Left wing (icon + label — new)

| Property | Value | Applies to |
|----------|-------|------------|
| Layout | `HStack(spacing: 4)` — icon then label, replacing today's icon-only left wing | Both Bluetooth and Charging |
| Icon size | `.font(.system(size: 13, weight: .semibold))` on the `Image(systemName:)` — new explicit size (today relies on the SF Symbol default); pairs visually with the 12px label | Both |
| Label text | 12px semibold, `.system(size: 12, weight: .semibold, design: .rounded)`, line-height 1.2, `.foregroundStyle(.white)` | Both |
| Label visibility | **Only shown in the positive state** — "Charging" only while `isCharging == true`; "Connected" only while `isConnected == true`. Negative states (disconnected / not charging / full) stay **icon-only, no label** (D-04) — same 50%-opacity dimming Islet already applies today, unchanged. | Both |
| Left padding from notch edge | 12px — unchanged from today's existing `wings(for:)`/`deviceWings(for:)` left padding, reused verbatim | Both |
| Icon color (unchanged) | Charging bolt: `Color.green` while charging / `Color.white.opacity(0.6)` otherwise. Device glyph: `deviceAccent.opacity(iconOpacity)` (theme-selectable accent, dims to 0.5 opacity when disconnected) | Both — no change to existing icon-tint logic |

### Right wing (status indicator — restyled)

| Property | Value | Applies to |
|----------|-------|------------|
| Charging | **No change** — existing `BatteryIndicator(level: percent, accent: chargingAccent)` already renders green + percentage while charging, already matches the Droppy reference (D-02) | Charging only |
| Bluetooth, battery known | **No change** — existing `BatteryIndicator(level: battery)` stays as-is when the device reports a level (D-02) | Bluetooth only |
| Bluetooth, connected, no battery known | **NEW:** a fixed-green ring, `Circle().strokeBorder(Color.green, lineWidth: 1.5)`, diameter 14px, transparent fill — replaces today's `checkmark` glyph for this specific case | Bluetooth only |
| Bluetooth, disconnected | **No change** — existing `Image(systemName: "xmark")`, 13pt bold, `Color.white.opacity(0.5)` — D-04: negative states keep today's dimmed-icon-only precedent, not restyled | Bluetooth only |
| Right padding from notch edge | 14px — unchanged, reused verbatim | Both |

**Spacing exception:** the 14px right padding is a legacy hardware-anchored constant tied to the physical notch/camera-bump position (it mirrors the physical asymmetry of the camera housing, not a design choice) — exempt from the 4pt design-grid rule, unchanged from shipped code. Same treatment as the EQ-01 1pt bar-width exception below: this value is preserved verbatim per `36-CONTEXT.md`/existing code, not adjusted to the nearest grid multiple. The 12px left padding is already grid-compliant and needs no such exception.

**Color rule (D-03, locked):** the right-wing positive-state indicator (green ring / green battery) is **fixed `Color.green`**, never `deviceAccent`/`chargingAccent`/theme-tinted — reads as a universal "active/good" signal independent of the user's chosen accent color. This does not change the LEFT icon's existing accent-tinted behavior.

**Reference match:** exact wording/colors verified against the user-supplied screenshots — `reference-droppy-airpods-pill.png` ("Connected" + green ring) and `reference-droppy-volume-charging-pills.png` ("Charging" + green battery/64%). Do not pull layout assumptions from `.planning/research/inspiration/notes.md`'s image numbering (confirmed unreliable in `36-CONTEXT.md`).

## Equalizer Motion Contract (EQ-01)

Target values extracted directly from the Skiper25 source (`reference-skiper25-equalizer.md`), adapted to preserve Islet's idle-CPU gate:

| Property | Today | New (locked) |
|----------|-------|---------------|
| Bar count | 5 | 5 — **unchanged** |
| Bar width | 2.5pt | **1pt** — sub-4pt-grid element size, exception noted below |
| Bar shape | `Capsule()` | **unchanged** — still `Capsule()` |
| Gap between bars | 2pt | **4pt** (fits the standard `xs` spacing token) |
| Height range | per-bar random 3–6pt (low) / 10–16pt (high) | **fixed 4–14pt for every bar** — no more per-bar randomized low/high |
| Box height (container) | 16pt | **unchanged**, still comfortably contains the new 4–14pt range, center-anchored |
| Color | `tint` param, currently passed `nowPlayingAccent` (theme-selectable) | **solid `Color.white`, fixed** — no gradient, no accent tint. Drop the `tint: nowPlayingAccent` argument at both call sites (`mediaWingsRow` ~L2001, `mediaExpanded` ~L2191); the struct's own default (`var tint: Color = .white`) already does the right thing once the argument is removed. |
| Motion (the core visual change) | continuous per-bar sine wave, independent period (0.55–1.05s) + phase per bar | **periodic reroll + spring**: every ~100ms while playing, reroll ALL 5 bars to new random target heights simultaneously (uniform `random(in: 4...14)`), each bar springs to its new target — snappier/more percussive than today's smooth wave. Source values (framer-motion units): `stiffness: 300, damping: 10` — closest SwiftUI analog is a fast, low-bounce spring (e.g. `.spring(response: ~0.25, dampingFraction: ~0.7)`); tune this on-device, it is a translation, not a 1:1 port (planner/executor discretion per `36-CONTEXT.md`) |
| Paused state | flat, per-bar randomized low height (3–6pt each) | **flat, ALL bars snap to the same fixed 4pt height** — matches the source's uniform `Array(bars).fill(0.1)` behavior (a single low value, not per-bar variety) |
| Idle-CPU gate (D-08, non-negotiable) | `TimelineView(.animation(paused: !isPlaying))`, zero clock while paused | **must be preserved exactly** — whatever reroll mechanism replaces the sine formula (e.g. a periodic `@State` target-height reroll driving `.animation(.spring(...), value:)`) must stop entirely (no `Timer`, no running `TimelineView`) the instant `isPlaying` goes false. This is the single most important carry-over constraint in this phase. |

**Spacing exception:** the 1pt bar width is a visual-element size, not a spacing gap — same precedent Phase 26 already established (icon sizes, stroke widths sit outside the 4pt grid; only gaps/padding must be on-grid). The 4pt bar gap itself IS on-grid.

**Scope guard (D-06, locked):** Skiper25 is a play/pause toggle button in its source form — EQ-01 is passive-visualizer-only. Do not make the bars tappable or wire them to any playback control; that's explicitly deferred (see `36-CONTEXT.md` Deferred Ideas).

## Signature Animation Contract (ONBOARD-04)

**Text (D-09, locked):** `"Meet Islet"` — replaces today's plain `Text("Meet Islet")` at 20px semibold. The body copy directly below it (`"Your notch, upgraded. Now Playing, charging, and a drag-and-drop shelf — always one glance away."`) is **byte-identical, unchanged** (D-13) — same 12px regular secondary text, same position, same gap.

**Mechanism (D-10, locked contract; exact API is executor's technical call per `36-CONTEXT.md` Claude's Discretion):** a per-character stroke-reveal, mimicking hand-drawn signing:
1. Extract each character's vector glyph outline from the signature font (Core Text's `CTFontCreatePathForGlyph` is the natural SwiftUI-native analog to the reference's `opentype.js` glyph extraction — confirm feasibility during technical research, not locked here).
2. Render each glyph's outline as a SwiftUI `Path`, animate `.trim(from: 0, to: progress)` from 0→1 (SwiftUI's native analog to the reference's `pathLength` 0→1) — a thick, round-capped stroke that visually "draws" the letter.
3. Once a glyph's stroke trim completes, reveal (or immediately show) that glyph filled solid — matching the reference's mask-then-fill technique.

| Property | Value |
|----------|-------|
| Per-character duration | 1.5s | 
| Per-character stagger delay | 0.2s × character index (character 0 starts at 0s, character 1 at 0.2s, etc. — spaces count as a character/index slot even though they render no visible stroke) |
| Easing | ease-in-out (`Animation.easeInOut(duration: 1.5)`) |
| Total animation length | ~3.3s for "Meet Islet" (10 characters incl. space; last character starts at 1.8s + 1.5s duration) — this is the expected, locked-by-reference result, not a bug |
| Stroke width (reveal mask) | ≈ 6pt (`fontSize × 0.22` from the reference, at fontSize 28 ≈ 6.16pt), round line caps/joins |
| Final fill color | `Color.orange` — solid, once each glyph's stroke completes |
| Trigger | Plays on the Welcome step's appearance (`.onAppear`); since onboarding shows once per install (persisted flag), no special "already played" guard is required — a rare Back-then-forward re-visit replaying the animation is harmless and not worth extra state |
| Repeat | Play once, no loop (`once: true` in the reference) |

**Color (D-11, locked — read carefully):** `Color.orange`, the **fixed SwiftUI literal** — this is NOT the user's theme-selectable accent (`ActivitySettings.palette` = `[.white, .blue, .green, .orange, .pink, .purple]`, default index 0 = **white**). Do not wire this to `nowPlayingAccent`/`chargingAccent`/`deviceAccent` or any environment accent key — it must render orange regardless of what accent color the user has chosen in Settings. This is a scoped, literal exception, same category as the fixed-green HUD indicators above.

**Font (D-12 — FLAGGED, resolved by this research pass):**

The reference's `LastoriaBoldRegular.otf` ("La storia" by Abo Daniel, 2019) is confirmed via web search to be the **demo/free version, personal-use-only** — a commercial license requires a separate purchase (via Creative Fabrica). Islet is a paid €7.99 product; shipping the demo `.otf` as-is would be a license violation, not just an unconfirmed risk.

**Locked default substitute:** **Dancing Script, Bold weight** (SIL Open Font License 1.1, Google Fonts, by Impallari Type) — confirmed OFL, explicitly permits commercial embedding/redistribution. Casual bounce-and-vary script style is the closest same-genre substitute available under a real commercial-safe license.

| Property | Value |
|----------|-------|
| Font | Dancing Script — Bold static weight |
| Font size | 28pt (up from today's 20px semibold system text — script fonts read visually smaller at equivalent point size than San Francisco; this is a new, larger reserved heading region, not a copy-paste of the old size) |
| Heading region reserved height | ~56pt (was one 20px text line + 8px gap) — the Welcome step has the least other content of the 4 onboarding steps, and `onboardingSize` (420×320, vertically-centered content per `26-UI-SPEC.md`) has headroom for this |

**New build touchpoint (flag for planner):** this is the first custom (non-system) font in this codebase — `project.yml` and the Xcode project currently register zero fonts. Bundling Dancing-Script-Bold.ttf requires adding the font resource + an `ATSApplicationFontsPath`/"Fonts provided by application" Info.plist entry (or runtime `CTFontManagerRegisterFontURLs`) — a new, small build-config change, not just a code change.

**If the user later wants the exact Lastoria Bold look:** purchase the Creative Fabrica commercial license for "La storia" first, then swap the font file only — the stroke-reveal mechanism and all other values above are font-agnostic.
