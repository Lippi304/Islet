<objective>
Add SHELF-06 (drag a shelf item back out to Finder or any other app) as a single vertical slice:
a pure missing-file gate (mirrors Phase 20's `shouldOpenShelfItem`), an `.onDrag` drag source on
`ShelfItemView` using `NSItemProvider(contentsOf:)` (D-04 default preview, D-02 silent no-op on a

for the duration of the drag and guarantees release via a bounded safety-net timer (D-03). D-01
(copy semantics — the drag never removes the item) falls out for free: no task in this plan calls
`ShelfCoordinator.remove`/`clear`.

Purpose: Prove the lower-risk drag-OUT direction end-to-end before Phase 22's higher-risk drag-IN
(click-through panel receiving external drags) is attempted.

never gets stuck open; a missing backing file degrades gracefully.

## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| `ShelfItemView.onDrag` → OS Drag Manager → receiving app (Finder/other) | Outbound only: an already-validated local file (`item.localURL`) is handed to the OS drag pipeline for another app to read. Nothing untrusted crosses INTO Islet through this boundary. |
| External file-system mutation (user/OS deletes the backing temp file) → the `fileExists` check inside `.onDrag` | A local, single-user TOCTOU-adjacent race between the existence check and `NSItemProvider(contentsOf:)` reading the file. |
