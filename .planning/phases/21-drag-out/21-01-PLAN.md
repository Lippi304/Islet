Add SHELF-06 (drag a shelf item back out to Finder or any other app) as a single vertical slice:
a pure missing-file gate (mirrors Phase 20's shouldOpenShelfItem), an .onDrag drag source on
ShelfItemView using NSItemProvider(contentsOf:) (D-04 default preview, D-02 silent no-op on a
vanished file), and a drag-pin lifecycle in NotchWindowController that suppresses grace-collapse
for the duration of the drag and releases it via BOTH a best-effort early-release signal (a global
.leftMouseUp monitor, mirroring the existing Pattern 1 .mouseMoved monitor) AND a guaranteed
20s safety-net timer (D-03). D-01 (copy semantics — the drag never removes the item) falls out for
free: no task in this plan calls ShelfCoordinator.remove/clear.

Purpose: Prove the lower-risk drag-OUT direction end-to-end before Phase 22's higher-risk drag-IN
(click-through panel receiving external drags) is attempted.
Output: A shelf item can be dragged out of the expanded island onto Finder/another app; the panel
resumes normal hover/grace-collapse promptly once the drag ends and never gets stuck open; a
missing backing file degrades gracefully.
