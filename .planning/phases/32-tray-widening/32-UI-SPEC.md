# Phase 32 — UI Design Contract

> Visual and interaction contract for Tray Widening. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Platform note:** This is a native macOS Swift/SwiftUI app (`NotchPillView.swift`), not a web/React project. The shadcn gate and CSS-token conventions in the template don't apply — there is no `components.json`/Tailwind config to detect. All values below are the project's own established SwiftUI conventions (SF Symbols, named `static let` size constants, the Phase 25 gradient material), extended for this phase's geometry-only scope:

1. `traySize`/`trayContentHeight` — new wider + shorter content box for `trayFullView` (D-01/D-02/D-03/D-06/D-08)
2. `ShelfItemView` icon + filename caption sizing (D-04) — a **global** change (single shared view, all 4 tabs)
3. `shelfRow` item spacing / clear-all icon — proportional bump to match the larger tiles (Claude's discretion)
4. No color, material, typography-family, or copy changes anywhere in this phase

## Layout & Interaction Contract

*(Supplementary section — this phase's actual design surface is a geometry change across 4 extension points, not a token table alone. Executor: read this section before touching code — it names the exact trap this phase's own research flagged.)*

### 1. `traySize` / `trayContentHeight` constants (new, `NotchPillView.swift`, next to `expandedSize`/`onboardingSize`)

```swift
static let traySize = CGSize(width: 840, height: 144)   // width per D-03; height field unused directly (see trayContentHeight below), kept for symmetry with expandedSize/onboardingSize's CGSize convention
static let trayContentHeight: CGFloat = 120   // D-06/D-08: single shared constant for both empty and non-empty states (per Open Question 2's recommendation — sized to the taller case, not two constants)
```

### 2. `blobShape`'s height ternary MUST change (Pitfall 1 — the load-bearing fix)

`trayFullView` passes `showSwitcher: true` (needed for the switcher row). Today's ternary at `blobShape`'s `baseHeight` line hard-overrides any `height:` override whenever `showSwitcher: true`:

```swift
// current (line ~1100) — height: override is silently ignored when showSwitcher is true
let baseHeight = showSwitcher ? Self.switcherContentHeight : (height ?? Self.expandedSize.height)
```

Must become:

```swift
let baseHeight = height ?? (showSwitcher ? Self.switcherContentHeight : Self.expandedSize.height)
```

Without this one-line change, `trayContentHeight` has **zero visual effect** and Tray silently stays at the old 196pt box. This is not optional polish — it is the mechanism that makes D-06 possible at all.

### 3. `trayFullView`'s `blobShape` call gets both overrides

```swift
blobShape(topCornerRadius: 24, bottomCornerRadius: 32, alignment: .top,
          width: Self.traySize.width, height: Self.trayContentHeight,
          shelfItems: [], shelfVisible: false, showSwitcher: true) { ... }
```

### 4. Outer `body` `.frame()` needs a third branch (Pattern 2 — do not skip)

Add `private var isTrayPresentation: Bool` (mirrors `isOnboardingPresentation`'s exact shape) and extend the width/height ternary at `body`'s outer `.frame()` (currently only branches `isOnboardingPresentation`) to also use `Self.traySize.width` / `Self.trayContentHeight` when Tray is active. This exact bug shape ("blobShape grew but the outer container frame didn't") has already shipped twice in this codebase (Phase 21 shelf, Phase 26 onboarding round 1) — do not reintroduce it a third time.

### 5. Panel-frame union (AppKit, `NotchWindowController.positionAndShow()`) — mandatory, not optional

Add a `trayFrame` union member sized to `CGSize(width: NotchPillView.traySize.width, height: NotchPillView.trayContentHeight + NotchPillView.switcherRowHeight)` and fold it into `panelFrame = expandedFrame.union(wings).union(onboardingFrame).union(trayFrame)`. Skipping this clips the 840pt content to the old ~420pt panel edge on the real screen (invisible in Xcode Previews — Pitfall 2).

### 6. `visibleContentZone()` branch (AppKit) — mandatory, CR-01/CR-02 discipline

Add a third branch (alongside the existing `isOnboardingActive` one) for `case .trayExpanded`, using `CGSize(width: NotchPillView.traySize.width, height: NotchPillView.trayContentHeight + switcherHeight)`. This must land in the **same commit** as items 3-5, then be verified via the mandatory on-device hover→expand→move-down trace (ROADMAP success criterion 4) — this is empirically not caught by unit tests alone in this codebase's history.

### 7. `ShelfItemView` — icon + caption (global change, all 4 tabs share this type)

```swift
Image(nsImage: NSWorkspace.shared.icon(forFile: item.localURL.path))
    .resizable()
    .frame(width: 40, height: 40)   // was 28×28 — D-04, GLOBAL: this is the one shared ShelfItemView, currently only rendered on Tray (TRAY-01 gates it off elsewhere), but the type itself changes for all 4 tabs
Text(item.filename)
    .font(.system(size: 9))
    .foregroundStyle(.secondary)
    .lineLimit(1)
    .truncationMode(.middle)
    .frame(maxWidth: 56)   // was 44
```

Per this phase's own RESEARCH.md (Assumption A1): D-04's wording has no Tray-only qualifier, so this is treated as an explicit, intended global change — not silently decided. Blast radius today is zero on Home/Calendar/Weather since `shelfStripVisible` gates them off (TRAY-01); this note exists so a future relaxation of that gate doesn't inherit an undocumented size bump.

### 8. `shelfRow` — item spacing + clear-all icon (Claude's discretion, proportional bump)

```swift
HStack(spacing: 14) {   // was 10 — more breathing room for the larger 40pt tiles
    ...
    Button(action: onShelfClearAll) {
        Image(systemName: "trash")
            .font(.system(size: 16))   // was 14
            .foregroundStyle(.white.opacity(0.7))
    }
}
.padding(.horizontal, 16)   // unchanged
```

### 9. Layout shape — unchanged (D-05, locked)

`shelfRow`'s `ScrollView(.horizontal) + HStack` structure is NOT replaced with a grid. The wider panel simply fits more 40pt tiles before scrolling kicks in.

### 10. Switcher-row Y-position divergence — EXPECTED, not a bug (D-07)

After the height shrink, the switcher icons (Home/Tray/Calendar/Weather) sit higher on Tray (120pt content box) than on Home/Calendar/Weather (196pt `switcherContentHeight` box). **This is the explicitly-accepted trade-off from Phase 32's discussion — do not "fix" it by reverting to the shared height constant.** Verification passes should note "switcher position differs on Tray — this is D-07, confirmed intentional" rather than flagging it as a regression.
