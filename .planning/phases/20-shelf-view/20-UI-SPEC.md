# Phase 20 — UI Design Contract

> Visual and interaction contract for the shelf strip appended to the expanded island. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Context:** This is a native macOS SwiftUI overlay (a 360×144pt "Dynamic Island" blob), not a web app. There is no shadcn/Tailwind/component-library design system — `components.json`/`tailwind.config.*`/`package.json` do not exist in this repo, and the project is not React/Next.js/Vite, so the shadcn initialization gate does not apply. All tokens below are derived from `Islet/Notch/NotchPillView.swift`'s existing, established conventions (already-shipped `mediaExpanded`/`expandedIdle`/wings/toast code) so the shelf row reads as part of the same UI, not a bolted-on component.

## Spacing Scale

This project does not use a web 8-point grid. `NotchPillView.swift` already uses a hand-tuned point scale sized to fit the fixed 360×144pt expanded blob (existing values in the file: 2, 4, 6, 10, 12, 14, 16, 19, 22, 24, 28, 32 — not all multiples of 4/8, deliberately fitted to available notch real estate). Phase 20 reuses this established scale rather than introducing a new grid.

Declared values for this phase's shelf row:

| Token | Value | Usage |
|-------|-------|-------|
| icon-gap | 2px | Between file icon and filename label (vertical stack) inside `ShelfItemView` |
| item-gap | 10px | Horizontal `HStack` spacing between shelf items (matches RESEARCH.md's `shelfRow` code example) |
| row-padding | 16px | Horizontal padding on the `ScrollView(.horizontal)` content — matches existing `.padding(.horizontal, 16)` used by `mediaExpanded`/`expandedIdle`'s own top-level HStacks |
| trash-hit-inset | 4px | Overlay inset for the per-item trash `Button` so its tap target clears the icon's edge without overlapping the filename |

Exceptions: none — no new spacing grid introduced; all four tokens reuse magnitudes already present elsewhere in `NotchPillView.swift`.

## Typography

This phase introduces exactly **one** new text role (the shelf item's filename caption). No new headings/body/display text is added — the shelf row is icons + one small caption + SF Symbol buttons.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Filename caption (new, this phase) | 9px | regular (400) | N/A — single line, `.lineLimit(1)` + `.truncationMode(.middle)`, never wraps (V5 mitigation for untrusted filenames, per RESEARCH.md Security Domain) |
| Body | not used in this phase | — | — |
| Heading | not used in this phase | — | — |
| Display | not used in this phase | — | — |

Weights declared for this phase (2 total, per contract): **regular (400)** for the filename caption; **semibold (600)** for SF Symbol icon buttons (trash + delete-all), matching the existing `transportButton` helper's `.font(.system(size: 16, weight: .semibold))` convention (`NotchPillView.swift` line 626).

Font sizes declared for this phase (3, within the 3-4 range):
- 9px — filename caption
- 11px — per-item trash icon (`xmark.circle.fill`), matches the existing small-badge precedent (RESEARCH.md's `ShelfItemView` example)
- 14px — delete-all trash icon (`trash`), matches the existing device/battery glyph size (`NotchPillView.swift` line 416's `.font(.system(size: 13, weight: .bold))` / line 285's bolt icon)

## Color

This app is monochrome-first (black blob, white content) with one persisted user-selectable accent reserved for live status. Phase 20 introduces no new colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (surface) | `Color.black` | The blob fill itself (`NotchShape().fill(Color.black)`) — the shelf row sits inside this same shape, no separate background |
| Secondary (content) | `Color.white` / `.white.opacity(0.6–0.8)` | Filename caption uses `.foregroundStyle(.secondary)` (same grey as the existing artist-text convention, D-05 precedent: "never accent-tinted"); trash/delete-all icons use `.white.opacity(0.7–0.8)`, matching the existing device-checkmark/disconnect icon opacity range (`NotchPillView.swift` line 417) |
| Accent (10%) | `@Environment(\.activityAccent)` (user-configurable, persisted) | **Not used anywhere in the shelf row.** Reserved elsewhere for live status only: `ProgressBar` fill, `EqualizerBars` tint, connected-device glyph/checkmark, battery indicator. File icons, filenames, and trash buttons are inert utility controls — same category as the artist-text/transport-buttons that are explicitly never accent-tinted today. |
| Destructive | not applicable — no red/destructive token introduced | Per D-03 (instant delete-all, no confirmation, session-only content) and the app's established all-white icon-button convention (`transportButton`), the trash and delete-all icons stay plain white/opacity, matching every other button in the app. Introducing a new red destructive color for a one-tap, reversible-by-redrag, nothing-precious-destroyed action would contradict the "lightweight-utility feel" CONTEXT.md D-03 explicitly calls for. |

Accent reserved for: `ProgressBar` fill track, `EqualizerBars` bars, connected-device glyph, `BatteryIndicator` — **not** the shelf row (explicit exclusion, this phase).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — the shelf row has no labeled call-to-action button, only icon-only controls (file-type glyph = open, small `xmark.circle.fill` = remove-one, `trash` = remove-all) |
| Empty state heading | Not applicable — per D-01, the shelf row does not render at all when `items.isEmpty` (structural absence via `if hasShelf { shelfRow(...) }`), not a rendered "empty" message |
| Empty state body | Not applicable — see above; there is nothing to word because nothing appears |
| Error state | Not applicable — per D-04, a click on an item whose local file is gone is a silent no-op (`guard FileManager.default.fileExists(...) else { return }`); no dialog, no toast, no inline copy |
| Destructive confirmation | Per D-03: none. Delete-all clears instantly on tap, no confirmation dialog or copy |
| Accessibility label — per-item trash | `"Remove \(item.filename)"` — icon-only `Button`s need an accessible name; default not covered by RESEARCH.md's code sample, added here per accessibility-basics baseline |
| Accessibility label — delete-all | `"Clear shelf"` |
| Accessibility label — shelf item (open) | `"Open \(item.filename)"` |

## Layout Notes (implementation-facing, non-visual-token specifics)

For the planner/executor — supplements RESEARCH.md's Pattern 1/2 code, does not restate them:

- **`shelfRowHeight = 56pt`** (confirms RESEARCH.md's Assumption A3 seed value as final): box math = 28pt icon (matches `transportButton`'s established 28×28 touch-size convention, reused per ladder rung 2) + 2pt icon-gap + ~11pt caption line + ~7.5pt top/bottom padding × 2 ≈ 56pt. Use this exact constant in both `NotchPillView.shelfRowHeight` and `NotchWindowController`'s panel-union math (RESEARCH.md Pitfall 1) — single source of truth, not re-derived in two places.
- **File icon size: 28×28pt** — reuses `transportButton`'s existing frame size instead of inventing a new icon dimension (ladder rung 2: already-established pattern).
- **Per-item trash icon: 11pt, `.topTrailing` overlay** — small enough not to visually compete with the 28pt file icon, matches RESEARCH.md's `ShelfItemView` code example exactly.
- **Delete-all icon: 14pt, far-right of the `HStack`** — sized between the 11pt per-item trash and the 28pt file icon so it reads as a "row-level" action, not an item-level one.
- **`bottomCornerRadius: 20`** when the shelf row is present — already the established value used by `expandedIdle`/`mediaUnavailable` today (no new radius token needed).
