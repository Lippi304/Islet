<objective>
Close the CR-01 blocking gap from `20-VERIFICATION.md`/`20-REVIEW.md`: `syncClickThrough()`
currently sets `panel.ignoresMouseEvents` for the ENTIRE panel rectangle whenever
`interaction.isExpanded` is true, but `positionAndShow` reserves `NotchPillView.shelfRowHeight`
(56pt) in the expanded panel frame UNCONDITIONALLY, even though the visible black blob
(`NotchPillView.blobShape`) only grows into that space when `shelfItems` is non-empty. Since the
shelf is empty by default in every Release build today, this creates a permanent invisible 56pt
click-swallowing band under the expanded island for every user — a direct regression of the
documented click-through invariant (Pitfall 3 / D-07: "clicks outside the pill always pass
through").

**Revision note (this plan was revised after `/gsd:plan-phase` checker feedback):** the first
version of this plan closed CR-01 by making `positionAndShow`'s panel-height reservation
CONDITIONAL on item count and re-resolving the panel frame (a live `NSPanel.setFrame`) whenever
the shelf crossed the empty/non-empty boundary. The checker correctly flagged that this
reintroduces the exact "live panel resize during an active SwiftUI spring" hazard the file's own
Pattern 4/Pitfall 4 doc comment exists to prevent, specifically on the SHRINK-to-empty transition
(last item deleted / clear-all): the panel would hard-resize synchronously in the same runloop
tick that starts the 0.35s spring shrinking the visible blob, clipping the animated blob for the
whole spring duration. This revision instead adopts `20-REVIEW.md`'s alternative CR-01 strategy
(b): the panel stays **statically** sized at the max reservation (exactly as it is today, and as
it was before this phase) — there is NO live panel resize and NO animation race. Instead, the
**click-through hit-test** in `syncClickThrough()` is scoped to the actual visible blob rect
(computed the same way `NotchPillView.blobShape` computes its height), so the reserved-but-
invisible band never swallows a click regardless of the panel's static size.

While in this exact code region, also close the two non-blocking findings from the same review
(WR-01, WR-02): the shelf delete/clear-all mutations aren't wrapped in the controller's standard
spring animation (so the shelf row's fade and the blob's height change snap instantly instead of
animating), and the `shelfViewState.items = shelfCoordinator.logic.items` resync line is
duplicated verbatim across three handlers. Both are fixed by the same
`resyncShelfViewState(animated:)` helper this plan introduces.

Purpose: Make the click-through hit-test this phase's shelf-sizing math depends on actually
correct (not just wired) before Phase 21/22 build on top of it, since both later phases inherit
this same panel-reservation foundation — without introducing a new animation race in the process.
Output: `syncClickThrough()` scopes interactivity to the visible blob height only (never the full
static panel rect); all three shelf mutation paths funnel through one animated (or explicitly
non-animated) resync helper that also re-triggers the click-through hit-test recompute.
</objective>

// lines 650-671 — handlePointer(at:): the GLOBAL .mouseMoved handler. `activeZone` is
// expandedZone while expanded (the generous keep-open region), hotZone while collapsed.
// pointerInZone is the edge-tracked flag driving handleHoverEnter/handleHoverExit. This
// plan adds a `lastPointerLocation` store at the top and a `syncClickThrough()` call at
// the bottom (only while expanded) so the click-through hit-test re-scopes on every tick,
// not just at zone-crossing edges — because the visible-content boundary (which the shelf's
// item count toggles) is SMALLER than expandedZone and is never itself crossed by this
// function's existing enter/exit edge detection.
private func handlePointer(at point: CGPoint) {
    let activeZone = interaction.isExpanded ? (expandedZone ?? hotZone) : hotZone
    guard let zone = activeZone else { return }
    let inside = zone.contains(point)
    if inside && !pointerInZone {
        pointerInZone = true
        handleHoverEnter()
    } else if !inside && pointerInZone {
        pointerInZone = false
        handleHoverExit()
    }
}

// lines 710-722 — the CR-01 bug: syncClickThrough() sets ignoresMouseEvents for the WHOLE
// panel whenever pointerInZone OR interaction.isExpanded — never scoped to the actual
// visible blob height. This is the ONLY function this plan changes to fix CR-01 (the panel
// geometry itself, positionAndShow, is untouched per strategy (b)).
private func syncClickThrough() {
    let interactive = pointerInZone || interaction.isExpanded
    panel?.ignoresMouseEvents = !interactive
}
