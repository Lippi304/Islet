<objective>
Close the CR-01 blocking gap from `20-VERIFICATION.md`/`20-REVIEW.md`: `positionAndShow` currently
reserves `NotchPillView.shelfRowHeight` (56pt) in the expanded panel frame UNCONDITIONALLY, even
though the visible black blob (`NotchPillView.blobShape`) only grows into that space when
`shelfItems` is non-empty. Since the shelf is empty by default in every Release build today, this
creates a permanent invisible 56pt click-swallowing band under the expanded island for every user —
a direct regression of the documented click-through invariant (Pitfall 3 / D-07: "clicks outside the
pill always pass through"). Fix strategy (a) from `20-REVIEW.md`: make the reservation conditional on
`shelfViewState.items.isEmpty`, and re-resolve the panel frame whenever the shelf crosses the
empty/non-empty boundary.

While in this exact code region, also close the two non-blocking findings from the same review
(WR-01, WR-02): the shelf delete/clear-all mutations aren't wrapped in the controller's standard
spring animation (so the shelf row's fade and the blob's height change snap instantly instead of
animating), and the `shelfViewState.items = shelfCoordinator.logic.items` resync line is duplicated
verbatim across three handlers. Both are fixed by the same `resyncShelfViewState(animated:)` helper
this plan introduces — which is also the natural place to trigger the boundary-crossing re-resolve
CR-01 needs, since it is the one place all three mutation paths already converge.

Purpose: Make the panel-sizing math this phase introduced actually correct (not just wired) before
Phase 21/22 build on top of it, since both later phases inherit this same panel-reservation
foundation.
Output: `positionAndShow` reserves shelf height only when the shelf has items; all three shelf
mutation paths funnel through one animated (or explicitly non-animated) resync helper that also
re-triggers panel re-sizing.
</objective>
