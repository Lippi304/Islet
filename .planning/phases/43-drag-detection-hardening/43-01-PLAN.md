<objective>
Fix DRAG-01: the island's auto-expand / Quick Action picker currently force-triggers on ANY
`.leftMouseDragged` tick inside the accept zone — an ordinary click's incidental drag-wobble, a
Finder window being moved, or any non-file drag — because `recheckDragAcceptRegion()` gates the
arm branch purely on pointer geometry, never on whether a genuine external file drag is actually
in progress. `dragPasteboardChangeCount` is tracked today but compared to itself AFTER being
overwritten on the same tick, so it can never detect a change (root cause confirmed in
`43-CONTEXT.md` / `43-PATTERNS.md`).

Purpose: Close the false-trigger regression (reported since Phase 24/34) without adding
perceptible latency to the genuine drag-to-expand path (D-05), and without touching
`NotchInteractionState.swift`'s deliberately geometry-agnostic `.dragEntered` transition.

Output: A new pure gate function `isGenuineFileDrag` (with unit tests) plus the two-line
controller change that makes `dragPasteboardChangeCount` an actual per-gesture baseline instead
of a self-referential no-op.
</objective>

## STRIDE Threat Register

This phase is local-only drag-gesture/pasteboard gating logic in an existing macOS app — no
network, no auth, no user-supplied data beyond local `NSPasteboard` drag content already read by
pre-existing code (`fileURLs`, unchanged this phase). The threat surface is minimal by
construction; the register below documents that assessment explicitly rather than omitting it.

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|------------------|
| T-43-01 | Tampering | `NSPasteboard(name: .drag)` read in `isGenuineFileDrag`'s caller | accept | Same-machine, same-user pasteboard; a local process could pre-seed it with crafted file URLs, but this only affects what shows in Islet's OWN Quick Action picker (Drop/AirDrop/Mail) for that user's own session — no privilege escalation, no cross-user exposure. This phase does not change what data is trusted from the pasteboard, only WHEN the controller reacts to it. |
| T-43-02 | Denial of Service | `recheckDragAcceptRegion` / `handleDragApproachTick` | accept | Runs on the existing `.leftMouseDragged` global-monitor cadence (unchanged this phase); the new gate is a single pure boolean comparison, no added allocation or loop — no new performance/DoS surface. |
| T-43-03 | Information Disclosure | `isGenuineFileDrag`'s `urls` parameter | accept | Local file paths already flow through `fileURLs`/`pendingDrop` today (Phase 22/24, unchanged); this phase adds no new read of pasteboard content, only an additional boolean gate on data already being read. |

No Spoofing / Repudiation / Elevation-of-Privilege entries — this phase has no identity, no audit
log, and no privilege boundary anywhere in its scope.
