<objective>
Wire the launch-time onboarding gate into the real app: `AppDelegate` stops auto-opening Settings
on first launch and instead hands `isFirstLaunch` to `NotchWindowController.start(isFirstLaunch:)`,
which uses Plan 26-01's pure gate functions to decide -- once, at startup -- whether this session

calls (D-01) instead of firing them eagerly as it does today.

sequential wave) adds the actual step-advance/grant/settings-hop/finish behavior once this state
exists.

install so no existing user is ever forced through onboarding (RESEARCH.md Pitfall 2).
