<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 1: On-device Apple Music `current track`/`loved` matrix (library / streaming-only x play / pause)</name>
  <files></files>
  <action>No code — pure Terminal verification against real Music.app library/streaming state on this dev machine.</action>
  <what-built>
    Nothing built this task — this is a direct Terminal `osascript` sanity check of a
    documented, forum-corroborated Apple Music AppleScript bug (error -1728), scoped to this
    project's own hardware per D-05.
  </what-built>
  <how-to-verify>
    Run each of the following three commands in Terminal once per state combination below (4
    combinations total: library track playing, library track paused, streaming-only track
    playing, streaming-only track paused):
      `osascript -e 'tell application "Music" to get name of current track'`
      `osascript -e 'tell application "Music" to get loved of current track'`
      `osascript -e 'tell application "Music" to set loved of current track to true'`
    1. Play a track that IS in your local Music.app library. Run all three commands. Record
       the output of each (expected: succeeds, returns the track name / current loved boolean /
       sets loved without error).
    2. Pause that same library track. Repeat all three commands. Record the output.
    3. Play a track that is NOT yet added to your local library (a streaming-only "For You" /
       Listen Now / catalog track). Repeat all three commands. Record the output — RESEARCH.md
       predicts `error "Music got an error: Can't get name of current track." number -1728` for
       at least the first command in this state.
    4. Pause that streaming-only track. Repeat all three commands. Record the output.
    5. Report the exact output (success value, or exact error text + error number) for all 12
       command runs (3 commands x 4 states).
  </how-to-verify>
  <resume-signal>Reply with the exact output of all 12 command runs (or a clear PASS/-1728/other-error summary per state-combination x command), plus a one-line overall verdict: `matrix-confirms-1728-on-streaming-only` / `matrix-shows-different-behavior` (describe exactly what differed from RESEARCH.md's prediction).</resume-signal>
</task>
