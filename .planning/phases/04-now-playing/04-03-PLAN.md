<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| untrusted media metadata → SwiftUI Text | title/artist (from arbitrary apps, via Plan 02's model) are rendered in `Text`. SwiftUI `Text` does NOT interpret format strings or execute interpolation — there is no injection vector — but unbounded length could break layout. |
| untrusted artwork NSImage → Image | The pre-decoded `NSImage?` (nil if the bytes were invalid, per Plan 02) is shown via `Image(nsImage:)`; nil → placeholder. No raw byte handling here. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-09 | Tampering / DoS (layout) | Over-long or adversarial title/artist string | mitigate | Title + Artist render with `.lineLimit(1)` + `.truncationMode(.tail)` — bounded display, no overflow. No format-string interpolation (SwiftUI Text is inert to it). |
| T-04-10 | Denial of Service (resource) | Equalizer `.repeatForever` left attached when paused/idle → render loop never sleeps | mitigate | EqualizerBars passes a finite/`.default` animation (not `.repeatForever`) when `isPlaying` is false (Pattern 4 / Pitfall 5). Verified on-device in Plan 04 UAT (`sample Islet` / Energy = idle). |
| T-04-11 | Tampering | Nil/invalid artwork rendered as a broken image | accept | Nil artwork falls to the music.note placeholder; an invalid image already arrives as nil from Plan 02's wrapper decode. No crash path. Low risk. |
</threat_model>
