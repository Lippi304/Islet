<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| perl child lifecycle ↔ controller | The controller owns the monitor (which owns the child). Start in start(), tear down in deinit. A failure to tear down leaks a perl process. |
| live media IPC ↔ visibility/fullscreen gate | Media show/hide must pass through the single updateVisibility() so it inherits the Phase-2 fullscreen + Phase-1 clamshell hide; a second show/hide site would race them into flicker/ghost. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-12 | Denial of Service (resource leak) | Orphaned perl child if the controller doesn't tear down | mitigate | `deinit` calls `nowPlayingMonitor?.stop()` (→ `stopListening()`), mirroring `powerMonitor.stop()`. UAT step 8 verifies no leftover perl after quit. |
| T-04-13 | Denial of Service (resource) | Equalizer render loop alive while paused/idle | mitigate | The view (Plan 03) removes the repeatForever when not playing; UAT step 3 confirms idle CPU ~0% on-device (the only way to verify render-loop behavior — 04-VALIDATION.md). |
