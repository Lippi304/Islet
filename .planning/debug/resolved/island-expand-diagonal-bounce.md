## Resolution

root_cause: "ALL FOUR 'island' matchedGeometryEffect call sites in NotchPillView.swift (collapsedIsland, blobShape, wingsShape, mediaWingsOrToast) applied `.frame(...)` BEFORE `.matchedGeometryEffect(id: \"island\", in: ns)`. Two of the file's own doc comments ('Phase 15 architecture audit item 2' at :1399 and 'Finding 12' at :1591) already documented the CORRECT order (matchedGeometryEffect before frame) — the actual code had silently drifted to the wrong order at some point, and rounds 1 and 2 both 'fixed' their target call sites by matching them to the file's drifted (wrong) code convention instead of its own (right) documented convention, which is why neither produced any observable change on-device."
fix: "Round 3: reordered all four call sites so `.matchedGeometryEffect(id: \"island\", in: ns)` precedes `.frame(...)` — collapsedIsland, blobShape, wingsShape, and mediaWingsOrToast. This now matches what the file's own pre-existing doc comments always said the order should be. No visual/behavioral change to any view's own steady-state rendering — only the point where the shared-namespace anchor/size interpolation is captured."
