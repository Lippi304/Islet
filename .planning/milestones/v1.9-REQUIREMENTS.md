# Requirements Archive: v1.9 — Clipboard History

**Status:** ✅ SHIPPED 2026-07-23
**Defined:** 2026-07-22
**Core Value:** The notch becomes a beautiful, reliable "island" that shows now-playing media and reacts when you plug in the charger or connect a device — it must feel native, smooth, and as polished as the iPhone Dynamic Island.

Adds a menu-bar clipboard history (text + images, ~20-30 item cap, persistent across
relaunch/reboot, encrypted at rest, sensitive-content excluded) to Islet's existing status-item
menu, replacing the user's third-party CopyClip tool. Menu-bar-only — explicitly not a new
Island/notch view. Started 2026-07-22 while v1.4, v1.5, and v1.7 all remained open in parallel.

## Clipboard History

- [x] **CLIP-01**: User sees a menu-bar dropdown listing the last ~20-30 copied text/image items, oldest automatically evicted past the cap. **Outcome:** Shipped (Phase 58) — flyout submenu behind a "Clipboard History" anchor (D-15 revision from an inline top-of-menu list, for more visible entries at once).
- [x] **CLIP-02**: Clicking an entry copies it back to the system clipboard (no auto-paste). **Outcome:** Shipped (Phase 58) as specified.
- [x] **CLIP-03**: The first 10 entries are directly selectable via ⌘0-⌘9. **Outcome:** Shipped (Phase 58) via a hybrid `menuWillOpen`/`menuDidClose`-scoped NSEvent keyDown monitor, since submenu key equivalents don't fire while the submenu is closed.
- [x] **CLIP-04**: Clipboard history persists across app relaunch and system reboot. **Outcome:** Shipped (Phase 56) — `ClipboardFileStore` encrypted JSON index + image files under Application Support; on-device kill-and-restart confirmed identical reload.
- [x] **CLIP-05**: "Delete All History" clears the entire history, with a confirmation dialog. **Outcome:** Shipped (Phase 58) — top-level `NSMenuItem` beside the Clipboard History anchor, destructive-confirm dialog + real on-disk delete.

## Privacy & Security

- [x] **PRIV-01**: Copies marked sensitive/concealed (`org.nspasteboard.ConcealedType`/`TransientType`) are never captured. **Outcome:** Shipped (Phase 57) — `ClipboardMonitor` filters at capture time, verified on-device against a real concealed-type source.
- [x] **PRIV-02**: The persisted history is encrypted at rest (CryptoKit AES-GCM, key in Keychain) — defense-in-depth beyond the ConcealedType marker check, since research found at least one major password manager (Bitwarden) doesn't reliably set it. **Outcome:** Shipped (Phase 56) — device-only AES-256 key in Keychain, on-device confirmed unreadable ciphertext on real disk.
