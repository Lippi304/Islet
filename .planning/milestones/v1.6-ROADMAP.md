# Milestone v1.6: Liquid Glass & System HUD Suite

**Status:** ✅ SHIPPED 2026-07-19
**Phases:** 35-42
**Total Plans:** 43

## Overview

Give Islet an edgier "Liquid Glass" material look and a suite of new Droppy-style collapsed-state system HUDs, plus a new dual-activity display concept for when two top-priority activities are live at once. Sequenced material-first (every other visual feature renders inside it), then risk-ascending through the new-transient-case pattern (cosmetic restyles → drop-session chip → Focus Mode spike → Volume/Brightness spike), with the fully-independent Sparkle integration floating after Phase 35, and the architecturally novel dual-activity display sequenced last, after Calendar Countdown exists as a proven single-winner case to combine with Now Playing. Numbered from Phase 35 (not 34) to avoid colliding with v1.5's still-open, unarchived Phase 34 — v1.5 remained open in parallel throughout this milestone, not archived.

## Milestone Summary

**Key Decisions:**

- Liquid Glass material pivoted to SwiftUI's native `.glassEffect()` on macOS 26+ (D-20), keeping the custom Metal shader stack (D-01–D-19) as the `<26` fallback — a round-5 post-completion regression made the native path clearly the lower-risk long-term choice.
- Phase 37 (Drop-Session Summary Chip) abandoned rather than redesigned — its Tray-close trigger essentially never fires in real usage; HUD-07 dropped from the requirement set.
- Focus Mode detection uses `INFocusStatusCenter` Path A (confirmed via an on-device spike), not the Assertions.json/Full-Disk-Access fallback — but Path A itself required an undocumented Communication Notifications entitlement beyond authorization, found only via 38-09's actual functional state read.
- Volume/Brightness native OSD suppression was re-attempted and shipped in a gap-closure plan (39-08) via `.cghidEventTap`, reversing Phase 39's own initial spike finding that suppression was unreliable.
- The Update-available indicator was redesigned mid-phase from a collapsed-pill corner badge to a menu-bar status-item dot, after on-device UAT root-caused the badge's tap-dispatch bug to a click-through hot-zone gap — the same fragility class independently re-found and fixed in Phase 42.
- The dual-activity secondary bubble's interaction was redesigned live during on-device UAT from tap-to-expand/no-hover (locked D-12/D-13) to hover-reveal play/pause, by explicit user decision.

**Issues Resolved:**
