  <action>
    Remove the `@State private var profiles` property, the `makeProfiles()` factory, and the `height(_:at:)` sine method (D-07: replacing the per-bar `(low,high,period,phase)` model entirely). Add `static func targetHeight(bar: Int, bucket: Int) -> CGFloat` (keep `internal`, not `private` — same `@testable import` precedent `makeProfiles()` used) that combines `bar` and `bucket` through a `Hasher` (`hasher.combine(bucket); hasher.combine(bar)`), reduces `hasher.finalize()` via `abs(hasher.finalize()) % 1000` — the `abs()` is required, not optional: `Hasher.finalize()` returns a signed `Int`, and Swift's `%` preserves the dividend's sign, so a negative hash would otherwise produce a negative remainder and map below the `4...14` floor — into a `0..<1000` integer, and maps that linearly into `4...14` (`4 + Double(...) / 1000.0 * 10`). In `body`, keep the existing `TimelineView(.animation(paused: !isPlaying)) { context in let t = ... }` wrapper unchanged, but add `let bucket = Int(t / 0.1)` (D-07: ~100ms reroll interval) and render each bar as `Capsule().fill(tint).frame(width: 1, height: isPlaying ? Self.targetHeight(bar: i, bucket: bucket) : 4).animation(.spring(response: 0.25, dampingFraction: 0.7), value: bucket)` — the per-bar `.animation(value: bucket)` makes SwiftUI spring-interpolate every bar simultaneously each time `bucket` increments, producing the locked snappier reroll-and-spring feel (D-07). Change `HStack(alignment: .center, spacing: 2)` to `spacing: 4`. `boxHeight` (16) stays unchanged. Update the struct's header comment (currently describing the sine-profile approach) to describe the new bucket/hash mechanism, preserving its warning that `TimelineView(.animation(paused:))` must never be swapped for an unconditional `.repeatForever` clock (still the load-bearing idle-CPU invariant, D-08).
