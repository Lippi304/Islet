  <behavior>
    - At elapsed time 0 (view just appeared), no glyph shows any stroke or fill yet.
    - At elapsed time `0.2*i` to `0.2*i + 1.5` for character index `i`, that glyph's stroke progressively reveals (0→1) with an ease-in-out curve.
    - Once elapsed time passes `0.2*i + 1.5` for character index `i`, that glyph shows solid fill instead of a partial stroke.
    - Once elapsed time passes the full reveal duration (`0.2*(characterCount-1) + 1.5`, ~3.3s for "Meet Islet"), a real `@State` flip stops the `TimelineView`'s clock — not a one-shot wall-clock comparison recomputed once at body-construction time (that form would bake `paused: false` in forever, since nothing else re-invokes `SignatureHeading.body` once the reveal starts, and the clock would tick indefinitely even after the reveal visually completes).
  </behavior>
  <action>
    Add `@State private var appearedAt: Date?` and `@State private var isRevealComplete = false`, plus an `.onAppear { if appearedAt == nil { appearedAt = Date() } }` guard (D-10: "Plays on the Welcome step's appearance... a rare Back-then-forward re-visit replaying the animation is harmless," so no extra "already played" flag is needed beyond this per-view-instance state). Replace the current unconditional `Canvas { ... context.fill for every path ... }` from Plan 36-03 with `TimelineView(.animation(paused: appearedAt == nil || isRevealComplete)) { context in Canvas { canvasContext, size in ... } }`, where `totalDuration = 0.2 * Double(glyphPaths.count - 1) + 1.5` (matches D-10's ~3.3s for 10 characters). Gate the `paused:` parameter on the `isRevealComplete` `@State` property — a genuine state mutation that re-triggers body evaluation, so SwiftUI actually observes the new `paused: true` — never on a `Date().timeIntervalSince(appearedAt!) > totalDuration` expression evaluated inline in the gate itself (that value is frozen at whatever it was when the body was first constructed and never re-evaluates). Inside the `TimelineView` content closure, before the `Canvas`, compute `elapsed = appearedAt.map { context.date.timeIntervalSince1970 - $0.timeIntervalSince1970 } ?? 0` (mirrors `ProgressBar`'s existing Unix-epoch `timeIntervalSince1970` convention in this same codebase, not the 2001-epoch `timeIntervalSinceReferenceDate` `EqualizerBars` uses for its own unrelated sine clock — do not mix the two epochs). Immediately after computing `elapsed`, if `elapsed > totalDuration && !isRevealComplete`, dispatch `DispatchQueue.main.async { isRevealComplete = true }` (deferring the mutation to the next run-loop tick avoids "modifying state during view update" — the closure body itself must stay a pure read). Pass `elapsed` into the `Canvas` closure for the per-glyph math. For each glyph index `i`: `delay = 0.2 * Double(i)`; `raw = ((elapsed - delay) / 1.5).clamped to 0...1`; ease it via the standard smoothstep curve `eased = raw * raw * (3 - 2 * raw)` (the concrete ease-in-out substitute for D-10's `easeInOut` curve). If `raw >= 1` (stroke fully revealed): `canvasContext.fill(path, with: .color(.orange))` (solid glyph, D-10's "reveal filled solid" step). Else if `raw > 0`: trim the glyph's path via `path.trim(from: 0, to: eased).path(in: .zero)` (concrete SwiftUI technique: `Path` conforms to `Shape`, so `.trim(from:to:)` returns a shape whose `.path(in:)` — ignoring the `rect` parameter, since a concrete `Path`'s own `Shape` conformance already carries absolute coordinates — yields the correctly-trimmed sub-`Path`) and `canvasContext.stroke(trimmedPath, with: .color(.orange), style: StrokeStyle(lineWidth: 6.16, lineCap: .round, lineJoin: .round))` (stroke width `fontSize * 0.22` at fontSize 28 ≈ 6.16pt, per D-10). Else (raw <= 0): draw nothing for that glyph yet.
  </action>
