<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|--------------|
| Islet process <-> `~/Library/DoNotDisturb/DB/Assertions.json` (or `INFocusStatusCenter`) | Untrusted-shape external data / OS-mediated state crossing into `FocusModeMonitor.poll()` |
| Islet process <-> `x-apple.systempreferences:` URL scheme | Outbound-only, no data crosses back; the URL string itself must not be attacker-influenced |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-38-03-01 | Denial of Service | `FocusModeMonitor.poll()` JSON parse | mitigate | Every dictionary/array access is optional-chained; a malformed/absent file is treated as "no change, keep prior state" (no `onChange` call), never a crash — matches RESEARCH.md's explicit defensive-parsing requirement |
| T-38-03-02 | Denial of Service (self-inflicted) | Poll timer under a revoked-mid-session FDA grant | mitigate | Fixed 2.5s interval with 500ms leeway regardless of read outcome — a failing read never triggers a faster retry or backoff-less hammer loop; the timer's cadence is identical whether reads succeed or fail |
| T-38-03-03 | Elevation of Privilege | Full Disk Access is a broad, all-files grant, requested via `openFullDiskAccessSettings()` | mitigate | Never called automatically — only reachable from the Settings toggle's explicit user-initiated flip-to-on (D-02, wired in Plan 38-06), never at launch/onboarding; the explanation copy (38-UI-SPEC.md) states exactly what is read and that files are never modified/sent |
| T-38-03-04 | Tampering (URL scheme misuse) | `openFullDiskAccessSettings()`'s hardcoded URL string | accept | The URL is a compile-time string literal with zero user-controllable interpolation — no injection surface exists by construction |

</threat_model>
