<objective>
Implement the round-4 fix from `35-CONTEXT.md` D-16/D-17/D-18: mask the 3 chromatic-fringe passes and the trailing white-wash overlay in `liquidGlassEffectLayer` (`Islet/Notch/NotchPillView.swift`) to the SAME `liquidGlassEdgeOpacity` edge-falloff the frost layer already uses, so both layers only render within the narrow rim band established by Plan 35-09 (D-12/D-13/D-14) and never wash the frost layer's dark center toward grey.

Round 3's shipped fix (Plan 35-09: dark frost base + edge-opacity ramp) was rejected on-device (`35-UAT.md` Test 1 Round 3, "Es ist immer noch so komisch silbern und nichts in Richtung liquid glass."). Root cause, confirmed by code inspection: the 3 `.blendMode(.screen)` fringe passes and the trailing `Color.white.opacity(parameters.backgroundOpacity)` overlay composite across the WHOLE shape with no masking of their own — `.screen` can only lighten, so regardless of how dark the frost layer's center is tuned, these two layers push the entire surface (center included) toward a lighter, washed-out grey.

This plan is a narrow, mechanical fix: reuse the exact `liquidGlassEdgeOpacity` Metal function already shipped in Plan 35-06/35-09 (zero `.metal` file edits, zero new `LiquidGlassParameters` fields) by calling it with mask-only arguments (`edgeOpacity: 1.0, centerOpacity: 0.0` — full visibility right at the rim, fully hidden at the interior) and chaining that as a `.colorEffect` on the 3 fringe fills and the white-wash overlay, multiplying each layer's own alpha down to zero by the time it reaches the frost's dark center.

Purpose: Give Plan 35-12's on-device UAT checkpoint a build where the dark-center/narrow-rim contrast (D-12/D-13/D-14, `reference-transparency-target.png`) is finally visible, unmasked by a whole-surface lightening wash.
Output: `liquidGlassEffectLayer` gains a `liquidGlassRimMask(shape:size:parameters:)` helper and 4 new `.colorEffect(rimMask)` call sites (3 fringe fills + the white-wash overlay); `LiquidGlassShader.metal`/`LiquidGlassShader.swift` untouched.
</objective>

<interfaces>
From Islet/Notch/LiquidGlassShader.metal (unchanged by this plan — read-only reference for this plan's new colorEffect call sites):
[[ stitchable ]] half4 liquidGlassEdgeOpacity(float2 position, half4 color, float2 size, float topCornerRadius, float bottomCornerRadius, float borderWidth, float blurWidth, float edgeOpacity, float centerOpacity)
Body: `float t = liquidGlassEdgeFalloff(...); float alpha = mix(edgeOpacity, centerOpacity, t); return half4(color.rgb, color.a * half(alpha));` — `t` is 0.0 right at the shape boundary, 1.0 at the interior (identical falloff the base warp shader uses). The frost layer (NotchPillView.swift ~318-332) calls this with `parameters.edgeOpacity`/`parameters.centerOpacity` (D-13: low at edge, high at center — makes the FROST opaque toward center). This plan calls the SAME function with the INVERTED intent — `edgeOpacity: 1.0, centerOpacity: 0.0` — so the resulting mask is fully visible (1.0 multiplier) at the rim and fully invisible (0.0 multiplier) at the interior; multiplying this against a layer's own existing alpha (`parameters.fringeOpacity` or `parameters.backgroundOpacity`) confines that layer to the rim without changing the frost's own parameters at all.

From Islet/Notch/NotchPillView.swift (current, being modified by this plan — full current body reproduced for reference, lines 304-368):
`private func liquidGlassEffectLayer(shape: NotchShape, size: CGSize, parameters: LiquidGlassParameters) -> some View` — ZStack containing (back to front): warped `.ultraThinMaterial` backdrop (no colorEffect), frost `Self.gradientMaterial` fill with `.colorEffect(liquidGlassEdgeOpacity, edgeOpacity: parameters.edgeOpacity, centerOpacity: parameters.centerOpacity)`, then 3 fringe fills (`Color.red/green/blue.opacity(parameters.fringeOpacity)`, each `.distortionEffect(shaders.red/green/blue, maxSampleOffset:...)` then `.blendMode(.screen)`) — THIS PLAN inserts `.colorEffect(rimMask)` between each fringe's `.distortionEffect` and its `.blendMode(.screen)`. After the ZStack: `.saturation(parameters.saturation)`, then `.overlay(Color.white.opacity(parameters.backgroundOpacity))` — THIS PLAN changes this line to `.overlay(Color.white.opacity(parameters.backgroundOpacity).colorEffect(rimMask))` — then `.clipShape(shape)`, `.allowsHitTesting(false)`.
