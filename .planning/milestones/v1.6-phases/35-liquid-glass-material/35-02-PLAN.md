  <action>
    Create `Islet/Notch/LiquidGlassShader.metal` with a header comment citing D-01 and this file's role as the project's first Metal shader. Declare `[[ stitchable ]] float2 liquidGlassDistortion(float2 position, float2 size, float topCornerRadius, float bottomCornerRadius, float borderWidth, float blurWidth, float distortionScale)` returning a warped `float2` position, using this exact algorithm: (1) compute `effectiveRadius` as a linear blend between `topCornerRadius` and `bottomCornerRadius` based on `position.y / size.y` (0 at the top edge uses `topCornerRadius`, 1 at the bottom edge uses `bottomCornerRadius`) — approximates `NotchShape`'s asymmetric hanging-pill silhouette for the warp math without needing to replicate its exact quad-curve path; (2) compute a rounded-box signed distance to the shape boundary using the standard rounded-rect SDF (`q = abs(position - size * 0.5) - (size * 0.5 - effectiveRadius)`; `dist = length(max(q, 0.0)) + min(max(q.x, q.y), 0.0) - effectiveRadius`); (3) convert to an inward "edge closeness" `edgeDist = -dist` (positive inside the shape, growing toward the center); (4) compute `edgeSize = min(size.x, size.y) * borderWidth` (the width of the warp band, mirroring the reference's `edgeSize = min(actualWidth, actualHeight) * (borderWidth * 0.5)` but using the full `borderWidth` fraction directly since this port folds the reference's separate `brightness`/`opacity` center-tone controls into this one falloff curve — no separate `brightness` parameter, since the falloff already asymptotes to near-zero displacement in the interior, which is what a mid-tone `brightness` value produced in the original SVG map — see 35-UI-SPEC.md's Material/Shader Contract table note documenting this fold-in, kept in sync with this implementation); (5) compute the transition `t = smoothstep(0.0, edgeSize + blurWidth, edgeDist)` — 0 right at the boundary (full warp), 1 once `blurWidth` points past the border band (no warp), with `blurWidth` softening the transition exactly as the reference's `blur` parameter softens its displacement map; (6) compute an outward direction `dir = normalize(position - size * 0.5)` (guard the zero-vector case at dead center by adding a tiny epsilon, e.g. `1e-4`, before normalizing); (7) return `position + dir * (1.0 - t) * distortionScale`. Add inline comments mapping each step back to the reference's SVG filter graph step it replaces (displacement-map generation -> steps 1-2, blur -> step 5, distortionScale -> step 7) so a future reader can cross-reference `reference-GlassSurface.md` directly.
